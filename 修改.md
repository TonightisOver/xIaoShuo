# xIaoShuo - AI 中文小说生成平台

基于 LangGraph 和 DeepSeek 的智能小说创作平台。

## 功能特性

- 🤖 基于 DeepSeek 大模型的智能创作
- 📖 完整的小说创作流程：创意扩展 → 世界观构建 → 人物设计 → 大纲生成 → 章节生成
- 🔄 Human-in-the-Loop 人工审核机制
- ✅ 自动质量检查和评分
- 💾 状态持久化和断点续写

## 技术栈

- **LangGraph**: 工作流编排
- **LangChain**: LLM 集成
- **DeepSeek**: 大模型 API
- **FastAPI**: Web API 框架
- **PostgreSQL**: 数据持久化
- **SQLAlchemy 2.0**: 异步 ORM
- **Alembic**: 数据库迁移
- **Poetry**: 依赖管理
- **pytest**: 测试框架

## 快速开始

### 1. 环境要求

- Python 3.14+
- Poetry 1.8+
- PostgreSQL 15+
- DeepSeek API 密钥

### 2. 安装依赖

```bash
# 安装 Poetry（如果未安装）
pip install poetry

# 安装项目依赖
poetry install
```

### 3. 配置 DeepSeek API

#### 3.1 获取 API 密钥

1. 访问 [DeepSeek 平台](https://platform.deepseek.com)
2. 注册账号并登录
3. 进入 API Keys 页面
4. 创建新的 API 密钥

#### 3.2 配置环境变量

创建 `.env` 文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的配置：

```env
# 必需配置
DEEPSEEK_API_KEY=your-api-key-here

# 数据库配置
DATABASE_URL=postgresql+asyncpg://xiaoshuo:your_password@localhost:5432/xiaoshuo?ssl=disable
DATABASE_POOL_SIZE=5
DATABASE_MAX_OVERFLOW=10
DATABASE_ECHO=false

# 可选配置（以下为默认值）
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_TEMPERATURE=0.7
DEEPSEEK_MAX_TOKENS=2000
DEEPSEEK_TIMEOUT=120
DEEPSEEK_MAX_RETRIES=3

# 日志配置
LOG_LEVEL=INFO
LOG_FILE=logs/xiaoshuo.log
```

**重要提示**:
- `DEEPSEEK_API_KEY` 和 `DATABASE_URL` 是必需的
- 数据库设置详见 [DATABASE_SETUP.md](DATABASE_SETUP.md)
- 超时时间建议设置为 120 秒以处理复杂 prompt
- 日志文件会自动创建在 `logs/` 目录

#### 3.3 验证配置

运行部署验证脚本：

```bash
poetry run python verify_deployment.py
```

预期输出：
```
[SUCCESS] 部署验证通过！
```

### 4. 设置数据库

参考 [DATABASE_SETUP.md](DATABASE_SETUP.md) 完成 PostgreSQL 配置，或快速执行：

```bash
# 创建数据库和用户（在 psql 中）
CREATE DATABASE xiaoshuo;
CREATE USER xiaoshuo WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE xiaoshuo TO xiaoshuo;

# 运行数据库迁移
poetry run alembic upgrade head
```

### 6. 启动 Web API 服务

```bash
# 启动 API 服务器
poetry run python run_api.py

# 或使用 uvicorn 直接启动
poetry run uvicorn src.api.main:app --host 0.0.0.0 --port 8000

# 开发模式（自动重载）
poetry run uvicorn src.api.main:app --reload
```

API 服务启动后：
- API 文档: http://localhost:8000/docs
- ReDoc 文档: http://localhost:8000/redoc
- 健康检查: http://localhost:8000/api/v1/health

### 7. 运行测试

```bash
# 单元测试（不需要 API 密钥）
poetry run pytest tests/unit/ -v

# API 测试
poetry run pytest tests/api/ -v

# 集成测试（需要有效的 API 密钥）
poetry run pytest tests/integration/ -v

# 所有测试
poetry run pytest tests/ -v

# 生成覆盖率报告
poetry run pytest tests/ --cov=src --cov-report=html
```

### 8. 代码质量检查

```bash
# 代码格式检查
poetry run ruff check src/ tests/

# 类型检查
poetry run mypy src/

# 自动修复格式问题
poetry run ruff check src/ tests/ --fix
```

## 使用示例

### 方式一：通过 Web API（推荐）

```bash
# 1. 启动 API 服务
poetry run python run_api.py

# 2. 创建小说生成任务
curl -X POST "http://localhost:8000/api/v1/novels" \
  -H "Content-Type: application/json" \
  -d '{
    "idea": "一个程序员穿越到修仙世界",
    "novel_type": "玄幻",
    "target_words": 100000
  }'

# 响应示例
{
  "task_id": "novel-20260516-abc123",
  "status": "pending",
  "created_at": "2026-05-16T10:30:00",
  "estimated_duration_minutes": 70
}

# 3. 查询任务状态
curl "http://localhost:8000/api/v1/novels/novel-20260516-abc123"

# 4. 列出所有任务
curl "http://localhost:8000/api/v1/novels?status=completed&limit=10"
```

### 方式二：直接调用 Python API

```python
import asyncio
from src.core.langgraph.graph import create_novel_graph
from src.core.langgraph.state import NovelState

async def main():
    # 创建工作流图
    graph = create_novel_graph()
    
    # 初始化状态
    initial_state: NovelState = {
        "project_id": "novel-001",
        "idea": "一个程序员穿越到修仙世界",
        "novel_type": "玄幻",
        "target_words": 100000,
        "current_stage": "start",
        "chapters": [],
        "errors": [],
    }
    
    # 执行工作流
    result = await graph.ainvoke(initial_state)
    
    # 查看结果
    print(f"生成章节数: {len(result['chapters'])}")
    print(f"当前阶段: {result['current_stage']}")

if __name__ == "__main__":
    asyncio.run(main())
```

## API 文档

### 端点列表

#### 1. 健康检查
```http
GET /api/v1/health
```

响应：
```json
{
  "status": "healthy",
  "version": "1.0.0",
  "timestamp": "2026-05-16T10:30:00"
}
```

#### 2. 创建小说任务
```http
POST /api/v1/novels
Content-Type: application/json

{
  "idea": "小说创意（10-1000 字符）",
  "novel_type": "玄幻|仙侠|都市|科幻|历史|武侠|言情|悬疑",
  "target_words": 100000
}
```

响应（202 Accepted）：
```json
{
  "task_id": "novel-20260516-abc123",
  "status": "pending",
  "created_at": "2026-05-16T10:30:00",
  "estimated_duration_minutes": 70
}
```

错误响应：
- `400 Bad Request`: 验证失败（无效的小说类型）
- `422 Unprocessable Entity`: 请求格式错误（创意长度不符）
- `500 Internal Server Error`: 服务器内部错误

#### 3. 查询任务详情
```http
GET /api/v1/novels/{task_id}
```

响应：
```json
{
  "task_id": "novel-20260516-abc123",
  "status": "running",
  "progress": {
    "current_stage": "chapter_generation",
    "completed_stages": ["idea_expansion", "world_building"],
    "total_stages": 5,
    "percentage": 60
  },
  "created_at": "2026-05-16T10:30:00",
  "started_at": "2026-05-16T10:30:05",
  "estimated_completion": "2026-05-16T11:40:00",
  "result": null,
  "errors": []
}
```

状态值：
- `pending`: 等待执行
- `running`: 执行中
- `completed`: 已完成
- `failed`: 失败

#### 4. 列出任务
```http
GET /api/v1/novels?status={status}&limit={limit}&offset={offset}
```

参数：
- `status` (可选): 过滤状态
- `limit` (可选): 返回数量，默认 20，范围 1-100
- `offset` (可选): 偏移量，默认 0

响应：
```json
{
  "tasks": [
    {
      "task_id": "novel-20260516-abc123",
      "status": "completed",
      "created_at": "2026-05-16T10:30:00",
      "completed_at": "2026-05-16T11:35:00"
    }
  ],
  "total": 1,
  "limit": 20,
  "offset": 0
}
```

## 故障排除

### 1. API 密钥无效

**问题**: `AuthenticationError: Your api key is invalid`

**解决**:
1. 检查 `.env` 文件中的 `DEEPSEEK_API_KEY`
2. 确认 API 密钥有效且未过期
3. 访问 https://platform.deepseek.com 验证密钥
4. 清除可能冲突的环境变量: `$env:DEEPSEEK_API_KEY = $null` (PowerShell)

### 2. API 超时

**问题**: `Request timed out`

**解决**:
1. 增加 `DEEPSEEK_TIMEOUT` 配置（默认 120 秒）
2. 检查网络连接
3. 简化 prompt 长度

### 3. 日志文件权限

**问题**: `Permission denied: logs/xiaoshuo.log`

**解决**:
1. 确保 `logs/` 目录存在且可写
2. 或修改 `LOG_FILE` 配置到其他路径

### 4. 依赖安装失败

**问题**: `poetry install` 失败

**解决**:
1. 更新 Poetry: `pip install --upgrade poetry`
2. 清除缓存: `poetry cache clear pypi --all`
3. 重新安装: `poetry install`

更多问题请查看 [部署验证报告](.harness/changes/CHANGE-002-deepseek-api集成/10-部署验证报告.md)。
