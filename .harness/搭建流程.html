Claude Code + Harness 项目搭建流程

从打开 Claude Code 到可以用 Auto Mode 跑 harness 流程，只需做一次

1
打开 Claude Code，进入项目目录
一次性
①
启动 Claude Code
在项目根目录打开 Claude Code（CLI 或 IDE 插件均可）
②
确认工作目录正确
确保当前目录是项目根目录，Claude Code 会自动读取 CLAUDE.md
↓
2
搭建 Harness 体系
新项目一次性
①
运行 harness-setup
创建 .harness/ 目录结构、10 阶段流程规范、Skills、变更管理模板
/harness-setup
②
回答项目信息
技术栈、项目规模、需要的 Skills 等，skill 会生成定制化配置
③
确认方案，等待搭建完成
skill 自动创建所有文件，包括 CLAUDE.md（让 Claude 每次对话自动加载 harness 规则）
已有 harness 的项目跳过此步骤
↓
3
配置 Auto Mode 门禁
新项目一次性
①
运行 harness-auto
自动写入 .claude/settings.json 和两个 hook 脚本，把人工门禁从"软约束"升级为"硬约束"
/harness-auto
②
验证 hooks 已加载
查看已配置的 hooks 列表，确认两个门禁 hook 出现
/hooks
🔒 门禁 1 — 技术设计确认
写入阶段 3/4 文件前，检查：
· 02-技术设计.md 存在
· .gate1-approved 标记文件存在

确认架构后手动创建标记：
New-Item .harness/changes/CHANGE-XXX/.gate1-approved
🏁 门禁 2 — 交付确认
summary.md 标记"已完成"前，检查：
· 10-部署验证报告*.md 存在

无需手动操作，阶段 10 完成后自动解锁
↓
4
开启 Auto Mode，开始开发
每次对话
①
开启 Auto Mode
开启后 Claude 在批次内自动执行，不再每步询问权限
/auto
②
描述需求，启动流程
直接说需求，Claude 自动跑批次 A（阶段 1-2），完成后暂停等你确认架构
③
确认架构 → 创建 .gate1-approved → 继续
你说"确认"后，Claude 自动跑批次 B（阶段 3-10），中间不打扰你
④
确认交付，关闭变更
批次 B 完成后 Claude 呈现交付摘要，你说"确认交付"关闭变更
每次对话节奏（搭建完成后）
打开 CC
→
/auto
→
描述需求
→
等待架构方案
→
确认 + 创建 .gate1-approved
→
等待批次 B 完成
→
确认交付
/harness-setup 和 /harness-auto 只需在新项目里运行一次。之后每次对话只需 /auto，CLAUDE.md 会自动加载 harness 规则。
