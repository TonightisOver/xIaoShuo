# 需求分析

## 变更编号
CHANGE-005

## 变更标题
WebSocket 实时进度推送

## 需求背景
当前小说生成任务为异步执行，用户提交后只能轮询获取最终结果，无法感知中间进度。需要通过 WebSocket 实时推送生成进度，提升用户体验。

## 功能需求

1. 建立 WebSocket 连接，按 task_id 订阅任务进度
2. 推送事件类型：
   - `stage_start` — 阶段开始
   - `stage_complete` — 阶段完成
   - `chapter_progress` — 章节生成进度（含中间结果）
   - `error` — 错误通知
   - `completed` — 任务完成
3. 支持多客户端同时订阅同一任务
4. 连接保活：30 秒心跳

## 非功能需求

- 单任务支持多订阅者，互不阻塞
- 连接断开后自动清理资源
- 不影响现有 REST API 行为

## 验收标准

- WebSocket 端点 `/ws/tasks/{task_id}` 可正常连接并接收事件
- 生成过程中各阶段事件按序推送
- 心跳机制正常工作
- 所有测试通过（含 4 个新增 WebSocket 测试）
