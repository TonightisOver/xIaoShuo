# 专家评审报告 v1 - CHANGE-011 按卷生成

## 评审维度

### 架构设计

- volumes 表与 chapters 表通过 novel_id + volume_number 关联，结构清晰
- outline JSON 存储灵活，适合大纲结构多变的场景
- 按卷生成作为独立后台任务，不阻塞主流程

**评价: 合理**

### API 设计

- GET/PUT /projects/{id}/volumes/{num} 符合资源模型
- POST /projects/{id}/generate-volume 触发生成，语义明确
- 与现有 API 风格一致

**评价: 合理**

### 兼容性

- 旧格式 (chapter_outlines 无 volumes) 继续支持
- 新旧数据可共存，无迁移风险

**评价: 良好**

### 性能考量

- 按卷生成减少单次 LLM 调用的 token 量
- 前卷上下文传递保证连贯性，token 开销可控

**评价: 合理**

## 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 并发生成同一卷 | 低 | 状态字段可防重入 |
| outline JSON 格式不一致 | 低 | prompt 约束 + 解析容错 |

## 结论

设计合理，实现完整，向后兼容。批准合入。
