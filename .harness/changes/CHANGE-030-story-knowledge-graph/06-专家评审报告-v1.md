## 2. 安全性评审

### 2.1 Prompt 注入风险 — 中等

**位置**: `KG_EXTRACTION_PROMPT` 中 `{chapter_text}` 直接拼入 prompt。

**分析**: `chapter_text` 来源于数据库中已生成的章节内容（由 LLM 自身生成），非用户直接输入。但如果用户通过 API 手动编辑章节内容后触发抽取，恶意内容可进入 prompt。

**风险等级**: 低。攻击者需要先通过认证写入章节内容，且 prompt 注入的影响仅限于抽取结果质量（不涉及系统命令执行）。

**建议**: 可在 `chapter_text[:6000]` 截断前增加基础清洗（移除 `## 输出格式` 等可能干扰指令的标记），但非阻塞项。

### 2.2 SQL 注入风险 — 无

所有查询使用 SQLAlchemy ORM 构建，参数化绑定。`list_entities` 的 `name.contains(name)` 使用 SQLAlchemy 的 LIKE 绑定，安全。

### 2.3 数据隔离

所有查询均包含 `novel_id` 过滤条件，不存在跨项目数据泄露风险。

## 3. 代码质量

### 3.1 实体去重逻辑

**`_merge_entities` 方法分析**:

- 精确名称匹配 → 更新 `last_chapter`
- 别名匹配 → 复用已有实体 ID
- 无匹配 → 新建实体

**边界情况**:

| 场景 | 行为 | 评价 |
|------|------|------|
| 空名称 `""` | `continue` 跳过 | PASS |
| 同名不同类型 | 视为同一实体（仅更新 last_chapter） | 注意：可能导致人物和地点同名时误合并 |
| 别名与另一实体主名冲突 | 别名优先匹配 | 可接受 |
| LLM 返回重复实体 | 第一个新建，后续精确匹配更新 | PASS |

**建议**: 同名不同类型的情况在小说场景中罕见（如"长安"既是人名又是地名），当前实现可接受，后续可加 `(name, entity_type)` 联合去重。

### 3.2 三元组写入

- 正确跳过 subject/object 不在 `name_to_id` 中的三元组
- 未做重复三元组检测（同一章同一关系可能重复写入）— 可接受，因为每章只抽取一次

### 3.3 Feature Flag 关闭时的代码路径

| 入口 | flag=False 行为 |
|------|----------------|
| `extract_from_chapter` | 返回 `{entities_count: 0, triples_count: 0}` |
| `retrieve_context` | 返回 `""` |
| `check_consistency` | 返回 `[]` |
| `chapter_generation.node` | 跳过 KG 服务初始化 |
| `quality_check.node` | 跳过一致性检查 |
| API 端点 | 不检查 flag，直接调用服务（服务内部检查） |

**注意**: API 端点在 flag 关闭时仍可访问，但 extract 操作会返回空结果。这是合理设计（不阻塞 CRUD 操作）。

### 3.4 `_format_context` Token 裁剪

硬编码 1500 字符截断。中文约 750 token，合理。截断位置可能在句子中间，但作为 prompt 上下文可接受。

## 5. 发现的问题

### P2 — `extract-all` 端点无并发保护

`extract_all` 先清除所有数据再逐章重新抽取。如果两个请求并发执行，可能导致数据丢失。

**建议**: 加分布式锁或在路由层加 `novel_id` 级别的信号量。

### P3 — `list_triples` 返回类型未声明 response_model

`list_triples` 端点返回 `list[dict]` 但未声明 `response_model`，与其他端点风格不一致。功能正常但缺少 OpenAPI schema。

### P3 — `_get_recent_events` 中 `last_chapter` 可能为 None

查询条件 `KnowledgeEntity.last_chapter >= current_chapter - 3` 在 `last_chapter` 为 NULL 时，PostgreSQL 会排除该行（NULL 比较返回 NULL）。这意味着只设置了 `first_chapter` 但未更新 `last_chapter` 的事件实体不会被检索到。

**影响**: 首次出现的事件实体如果未被后续章节引用，不会出现在"最近事件"中。可接受但需知晓。
