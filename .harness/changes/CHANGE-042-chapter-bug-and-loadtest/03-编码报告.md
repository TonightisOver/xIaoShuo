## Bug 修复

### 修改文件：`src/api/services/novel_manager.py`

#### 修复1：`get_chapter` 补充 `novel_id` 字段（第 319 行）

| 项目 | 内容 |
|------|------|
| 方法 | `get_chapter(self, novel_id, chapter_number)` |
| 位置 | 约第 319 行，return 字典 |
| 变更 | 在 `"id": c.id` 后插入 `"novel_id": c.novel_id` |
| 原因 | `ChapterResponse` 模型要求 `novel_id` 为必填字段，缺失会导致 Pydantic 序列化失败 |

注：`get_chapter` 的 `.order_by(Chapter.id.desc()).limit(1)` 在本次修改前已存在（前次提交已修复），无需重复添加。

#### 修复2：`update_chapter` 查询加 `order_by/limit`（第 328-332 行）

| 项目 | 内容 |
|------|------|
| 方法 | `update_chapter(self, novel_id, chapter_number, **kwargs)` |
| 位置 | 约第 328-332 行，select 查询 |
| 变更 | 在 `where(...)` 后追加 `.order_by(Chapter.id.desc()).limit(1)` |
| 原因 | 同一 `novel_id + chapter_number` 存在多条记录时，`scalar_one_or_none()` 会抛 `MultipleResultsFound`；取最新一条（id 最大）符合业务语义 |

## 新建文件：`loadtest/`

| 文件 | 说明 |
|------|------|
| `loadtest/config.py` | HOST 从 `LOADTEST_HOST` 环境变量读取，默认 `http://localhost:8000`；从 `data/seed.json` 加载 `NOVEL_IDS` / `CHAPTER_NUMBERS` |
| `loadtest/data/seed.json` | 占位数据，需替换为真实 novel_id |
| `loadtest/tasks/__init__.py` | 空文件，使 tasks 成为 Python 包 |
| `loadtest/tasks/chapter_read.py` | `ChapterReadTasks`：随机取 novel_id + chapter_number，GET 章节，断言 200/404 |
| `loadtest/tasks/chapter_list.py` | `ChapterListTasks`：GET 章节列表（权重3）+ GET 小说详情（权重1） |
| `loadtest/tasks/chapter_generate.py` | `ChapterGenerateTasks`：POST 生成章节，body `{"start":1,"end":1}`，捕获 409 不算失败 |
| `loadtest/tasks/ai_rewrite.py` | `AIRewriteTasks`：POST 改写章节，`timeout=120` |
| `loadtest/locustfile.py` | `ReadHeavyUser`（ChapterRead:7, ChapterList:3，wait 0.5-2s）；`GenerateUser`（ChapterGenerate:1，wait 30-120s） |
| `loadtest/.gitignore` | 排除 `reports/` 目录 |

### 运行方式

```bash
# 安装依赖
pip install locust

# 启动（Web UI 模式）
locust -f loadtest/locustfile.py

# 无头模式（CI）
locust -f loadtest/locustfile.py --headless -u 50 -r 5 --run-time 60s

# 指定目标环境
LOADTEST_HOST=http://staging.example.com locust -f loadtest/locustfile.py
```

### 使用前准备

1. 将 `loadtest/data/seed.json` 中的 `"replace-with-real-novel-id"` 替换为数据库中真实存在的 novel_id
2. 按需调整 `chapter_numbers` 列表
