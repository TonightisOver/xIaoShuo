# 阶段 3: 编码计划 - CHANGE-032-live-knowledge-graph

## 1. 任务清单 (Tasks List)

### 任务 1: 数据库与迁移设计
- 目标: 创建 `knowledge_entity_states` 数据库表，并完成向后兼容的迁移。
- 修改文件:
  - `src/api/models/db_models.py`
  - 新增 Alembic 迁移脚本 `alembic/versions/20260520_add_knowledge_entity_states.py`
- 细节: 运行 `alembic upgrade head` 以在本地/Docker 数据库应用此更改。

### 任务 2: 知识图谱服务层逻辑升级
- 目标:
    1. 在 `KnowledgeGraphService._merge_entities` 中实现状态的继承合并。即：获取前一章节或初始 attributes 作为基础，将新抽取的 attributes 覆盖，保存最新全量状态至 `KnowledgeEntityState` 表并更新 `KnowledgeEntity.attributes`。
    2. 在 `KnowledgeGraphService.retrieve_context` 中重构：
       - 精准匹配大纲中提到的实体名称。
       - 获取上述实体在当前章节前的最新 `KnowledgeEntityState` 快照。
       - 从 `Character` 表读取角色的正式人设。
       - 检索这些实体在当前章节前的最新 active 三元组关系，去重保留最新演化，整合元数据（如好感度）。
       - 拼装高可读性、高一致性约束的 Markdown 上下文，注入到 Prompt 头部。
- 修改文件:
  - `src/api/services/knowledge_graph_service.py`

### 任务 3: API 路由升级
- 目标: 新增实体时空历史状态轨迹的查询接口。
- 接口: `GET /api/v1/projects/{novel_id}/knowledge-graph/entities/{entity_id}/history`
- 修改文件:
  - `src/api/routes/knowledge_graph.py`

### 任务 4: 测试编写与验证
- 目标: 覆盖状态继承合并、大纲上下文记忆匹配、恩怨去重检索。
- 新增文件:
  - `tests/unit/test_live_knowledge_graph.py`
- 动作: 运行 `pytest tests/unit/test_live_knowledge_graph.py` 以进行单元测试校验。

---

## 2. 依赖关系

1. **数据库迁移** 必须在 **服务层修改** 之前完成，否则代码运行将会因数据表缺失而报错。
2. **单元测试编写** 必须在 **服务层编码** 完成后进行，以便快速检测其语法与逻辑正确性。

---

## 3. 风险与回退方案
- **迁移失败**: 若本地 alembic 版本冲突或数据库无响应，使用 alembic downgrade，并检查 alembic.ini 和 src/core/database.py。
- **降级机制**: 若 LLM 抽取在部分特殊小说上性能欠佳，在 `retrieve_context` 增加 try-except 块，降级为默认的上下文生成机制，确保章节生成逻辑不受阻碍。
