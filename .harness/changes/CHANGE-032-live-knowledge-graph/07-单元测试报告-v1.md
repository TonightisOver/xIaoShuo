# 阶段 7: 单元测试报告 - CHANGE-032-live-knowledge-graph

## 1. 单元测试用例设计

针对活态知识图谱（Consistency & Memory）的三大核心特性，在 `tests/unit/test_live_knowledge_graph.py` 中设计并实现了三个极高仿真度的单元测试：

### 1.1 `test_state_inheritance_and_saving` (状态继承与快照写入测试)
- **目标**: 校验实体状态是否能够跨章节无缝继承，并在本章有新属性变化时执行合并与快照持久化。
- **校验点**:
  - 确认原实体在第 2 章的状态 `{location: "Beijing", status: "healthy"}` 正确加载。
  - 确认本章 LLM 抽取的变动 `{status: "injured", health: 60}` 完美继承合并为 `{location: "Beijing", status: "injured", health: 60}`（原有的 `location` 得到继承，`status` 被更新，`health` 新增）。
  - 确认实体主表属性 `last_chapter` 正确向前推演。
  - 校验 `KnowledgeEntityState` 表是否新增了一条第 3 章的全新历史状态快照。

### 1.2 `test_retrieve_context_active_relationship_deduplication` (关系时序去重测试)
- **目标**: 确保当两个实体在多章存在演进的关系时（如张三与李四好感度发生多次变化），检索模块仅提取最新的关系，剔除老旧历史关系。
- **校验点**:
  - 第 1 章确立关系：张三 ─(喜欢)─> 李四（好感度: 10）。
  - 第 2 章演化关系：张三 ─(喜欢)─> 李四（好感度: 30）。
  - 检索第 3 章上下文时，确认在 Prompt 中仅包含好感度为 30 的最新演变关系，完全排除好感度为 10 的旧记录。

### 1.3 `test_retrieve_context_outline_matching` (大纲高精度多重匹配测试)
- **目标**: 检验实体检索对大纲文本中的实体名称及别名的匹配能力。
- **校验点**:
  - 存在实体“张三”，其别名 aliases 为 `["小张"]`。
  - 大纲文本仅提及 `小张和李四一起商量对策。`
  - 检索判定“张三”（通过别名“小张”）与“李四”被双双精准锁定，而未在文本中出现的“王五”被安全忽略。
