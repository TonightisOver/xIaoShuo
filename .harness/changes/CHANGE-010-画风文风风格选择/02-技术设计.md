# 技术设计: 画风/文风风格选择

**变更 ID**: CHANGE-010  
**创建时间**: 2026-05-16

## 1. 设计概述

在小说项目中添加 `writing_style` 字段，创建时选择风格，生成时将风格描述注入到 LLM prompt 中，影响所有节点的输出风格。

## 2. 数据模型变更

### novels 表新增字段

```sql
ALTER TABLE novels ADD COLUMN writing_style VARCHAR(50) DEFAULT '现代白话';
```

### 风格定义（代码常量）

```python
WRITING_STYLES = {
    "轻松幽默": "请使用轻松幽默的文风，多用吐槽、梗和诙谐对话，节奏轻快活泼。",
    "热血燃向": "请使用热血燃向的文风，短句有力，战斗场面爽快激烈，节奏紧凑。",
    "细腻文艺": "请使用细腻文艺的文风，注重环境描写和心理刻画，语言优美有意境。",
    "史诗厚重": "请使用史诗厚重的文风，宏大叙事，气势磅礴，用词正式庄重。",
    "悬疑紧张": "请使用悬疑紧张的文风，善于营造氛围，短段落制造悬念，多设伏笔。",
    "古风典雅": "请使用古风典雅的文风，半文言半白话，多用四字词语和诗词典故。",
    "现代白话": "请使用通俗易懂的现代白话文风，贴近生活，自然流畅。",
    "暗黑压抑": "请使用暗黑压抑的文风，注重内心独白和环境渲染，氛围阴暗沉重。",
}
```

## 3. 影响范围

### 3.1 Prompt 注入点

风格描述作为系统指令添加到以下节点的 prompt 中：
- `idea_expansion` — 影响创意扩展的语言风格
- `world_building` — 影响世界观描述风格
- `character_design` — 影响人物描写风格
- `outline_generation` — 影响大纲叙述风格
- `chapter_generation` — **主要影响点**，直接决定正文风格

### 3.2 注入方式

在 NovelState 中传递 `writing_style`，各节点从 state 读取并拼接到 prompt：

```python
style_prompt = WRITING_STYLES.get(state.get("writing_style", ""), "")
prompt = f"{style_prompt}\n\n{original_prompt}"
```

## 5. 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 风格存储 | novels 表字段 | 简单直接，一对一关系 |
| 风格定义 | 代码常量 | 不需要动态管理，预设即可 |
| Prompt 注入 | 前置拼接 | 不修改原有 prompt 模板结构 |
| 默认值 | "现代白话" | 最通用，不影响现有行为 |
