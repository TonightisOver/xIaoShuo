# 技术设计

## 变更编号
CHANGE-008

## 架构概览

```
Internet → :80 Nginx → :8000 API (FastAPI + 静态前端) → :5432 PostgreSQL
```

## 镜像设计

采用 multi-stage build：

| 阶段 | 基础镜像 | 职责 |
|------|----------|------|
| Stage 1 | node:20-slim | 构建前端静态资源 |
| Stage 2 | python:3.14-slim | 运行 FastAPI 后端，携带前端产物 |

## 服务编排 (docker-compose.yml)

| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| db | postgres:17-alpine | 5432 (内部) | 带 healthcheck，数据挂载 volume |
| api | 本地构建 | 8000 (内部) | 启动时执行 alembic upgrade head |
| nginx | nginx:alpine | 80 (外部) | 反向代理 + WebSocket upgrade |

## Nginx 配置要点

- `proxy_pass http://api:8000`
- WebSocket upgrade: `Upgrade` / `Connection` 头转发
- 静态资源缓存头

## 数据持久化

- PostgreSQL 数据目录挂载到命名 volume `pgdata`

## 启动顺序

1. db 启动并通过 healthcheck
2. api 启动，执行数据库迁移
3. nginx 启动，开始接收流量
