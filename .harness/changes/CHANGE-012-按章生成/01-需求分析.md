# 需求分析: 按章生成

**变更 ID**: CHANGE-012  
**创建时间**: 2026-05-16  
**优先级**: P1

## 1. 需求背景

CHANGE-011 实现了按卷生成，但用户可能需要更细粒度的控制——只生成指定的几章。例如：
- 只重新生成第 3-5 章（对前面的不满意）
- 从第 10 章开始续写
- 只生成某一章的内容

**目标用户**: 需要精细控制生成范围的创作者  
**使用场景**: 部分章节需要重新生成或续写

## 2. 核心功能点

### 功能 1: 按章节范围生成
**描述**: 指定起始章节和结束章节，只生成该范围内的章节  
**API**: `POST /api/v1/projects/{novel_id}/generate-chapters`  
**参数**: `chapter_start` (int), `chapter_end` (int)

### 功能 2: 上下文自动注入
**描述**: 生成时自动将前面已有章节的内容作为上下文  
**实现**: 读取 chapter_start - 1 章的结尾作为 previous_chapter

### 功能 3: 前端章节范围选择
**描述**: 在小说详情页可以选择章节范围触发生成  
**位置**: NovelDetail 章节 Tab 中

## 3. 验收标准

- [ ] 可通过 API 指定章节范围生成
- [ ] 生成时自动注入前文上下文
- [ ] 已有章节被覆盖时标记为 "regenerated"
- [ ] 前端可选择章节范围
- [ ] 所有测试通过
