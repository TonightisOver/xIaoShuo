# CHANGE-043 需求分析：API 压力测试扩展

## 背景

项目已有基础 Locust 压测框架（CHANGE-042），覆盖 4 个场景：
- 章节读取 (chapter_read)
- 章节列表 (chapter_list)
- 章节生成 (chapter_generate)
- AI 改写 (ai_rewrite)

用户已完成 20 章 30 万字小说的数据准备，现需扩展压测覆盖面，同时支持 k6 作为备选工具。

## 需求描述

1. **扩展 Locust 压测场景**：覆盖项目全部核心 API 接口
2. **新增 k6 压测脚本**：提供 k6 版本的压测方案，支持 CI 集成和更精确的性能指标
3. **压测报告输出**：支持 HTML/JSON 报告生成

## 压测场景设计

### 用户画像（User Profiles）

| 画像 | 比例 | 行为模式 |
|------|------|----------|
| 读者 (Reader) | 60% | 高频读取章节、浏览列表、查看版本 |
| 作者 (Writer) | 25% | 编辑章节、保存、触发生成、查看任务 |
| 管理员 (Admin) | 15% | 浏览全局列表、查看图谱、管理世界观 |

### 压测目标

| 指标 | 目标值 |
|------|--------|
| 读接口 P95 延迟 | < 200ms |
| 写接口 P95 延迟 | < 500ms |
| AI 接口 P95 延迟 | < 60s（LLM 限制） |
| 错误率 | < 1% |
| 并发用户 | 50-200 |
| 持续时间 | 5-15 分钟 |

## 技术方案

### 方案 A：扩展 Locust（推荐）
- 优势：已有基础、Python 生态一致、Web UI 实时监控
- 新增场景模块 + 用户画像 + HTML 报告

### 方案 B：新增 k6 脚本
- 优势：更精确的性能指标、原生 CI 集成、阈值断言
- 输出 JSON summary + 可选 Grafana 集成

### 最终方案：A + B 并行
- Locust 用于交互式压测和探索
- k6 用于 CI 自动化回归和阈值守护
