## 4. 关键修复: NullPool

### 问题

初始测试运行时出现:
```
RuntimeError: Event loop is closed
```

### 原因

pytest-asyncio 在测试结束后关闭事件循环，但 asyncpg 连接池中的连接仍绑定在旧事件循环上，导致清理时报错。

### 解决方案

`tests/conftest.py` 中使用 NullPool:

```python
from sqlalchemy.pool import NullPool

engine = create_async_engine(TEST_DATABASE_URL, poolclass=NullPool)
```

NullPool 不缓存连接，每次操作创建新连接并立即释放，完全避免事件循环生命周期问题。
