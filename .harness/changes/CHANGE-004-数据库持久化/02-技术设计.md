# 技术设计: PostgreSQL 数据库持久化

**变更 ID**: CHANGE-004  
**创建时间**: 2026-05-16

## 1. 设计概述

将任务存储从 JSON 文件迁移到 PostgreSQL，使用 SQLAlchemy 2.0 async ORM + asyncpg 驱动 + Alembic 迁移管理。单表设计，JSONB 存储嵌套数据。

## 2. 系统架构

### 2.1 架构图

```
FastAPI (lifespan)
    ↓ init_db / close_db
Database Engine (AsyncEngine)
    ↓ async_sessionmaker
AsyncSession
    ↓
TaskManager (CRUD)
    ↓
PostgreSQL (tasks 表)
```

## 5. 核心代码设计

### 5.1 database.py

```python
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

engine: AsyncEngine | None = None
async_session: async_sessionmaker[AsyncSession] | None = None

async def init_db(database_url: str):
    global engine, async_session
    engine = create_async_engine(
        database_url,
        pool_size=5,
        max_overflow=10,
        pool_pre_ping=True,
    )
    async_session = async_sessionmaker(engine, expire_on_commit=False)

async def close_db():
    if engine:
        await engine.dispose()

async def get_session() -> AsyncGenerator[AsyncSession, None]:
    async with async_session() as session:
        yield session
```

### 5.2 db_models.py

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
from sqlalchemy import String, DateTime, JSON
from datetime import datetime

class Base(DeclarativeBase):
    pass

class Task(Base):
    __tablename__ = "tasks"
    
    id: Mapped[int] = mapped_column(primary_key=True)
    task_id: Mapped[str] = mapped_column(String(36), unique=True, index=True)
    novel_type: Mapped[str] = mapped_column(String(50))
    title: Mapped[str | None] = mapped_column(String(200))
    status: Mapped[str] = mapped_column(String(20), index=True)
    progress: Mapped[dict | None] = mapped_column(JSON)
    result: Mapped[dict | None] = mapped_column(JSON)
    errors: Mapped[list | None] = mapped_column(JSON)
    created_at: Mapped[datetime] = mapped_column(DateTime, index=True)
    updated_at: Mapped[datetime] = mapped_column(DateTime)
```

## 6. 关键设计决策

### 6.1 NullPool 解决事件循环冲突

**问题**: pytest-asyncio 和 asyncpg 连接池共享事件循环时产生冲突  
**方案**: 测试环境使用 NullPool，每次操作创建新连接  
**权衡**: 测试性能略降，但完全消除事件循环问题

### 6.2 JSONB 存储嵌套数据

**问题**: progress/result/errors 结构复杂且可变  
**方案**: 使用 PostgreSQL JSONB 类型存储  
**权衡**: 牺牲部分查询能力，换取 schema 灵活性

### 6.3 单表设计

**问题**: 当前只有 Task 实体  
**方案**: 单表设计，后续按需拆分  
**权衡**: 简单直接，避免过度设计

## 7. 风险和挑战

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| asyncpg 事件循环冲突 | 高 | NullPool（已解决） |
| 连接池耗尽 | 中 | pool_pre_ping + 合理配置 |
| Alembic 迁移冲突 | 低 | 单人开发，暂无冲突风险 |
