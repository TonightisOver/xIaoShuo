# 01-需求分析

**变更**: CHANGE-025  
**日期**: 2026-05-18

---

## 需求背景

用户反馈4个功能缺陷，影响核心创作流程。

---

## 功能点分析

### Bug 1: 概览创意修改不了
- **现象**: NovelDetail 概览tab中，创意(idea)、标题等字段只读展示，无法编辑
- **根因**: `NovelDetail.vue` overview tab 使用 `<dd>` 纯文本展示，无编辑入口
- **验收**: 概览tab可点击编辑创意/标题，保存后刷新显示新值

### Bug 2: 大纲章纲只有第一卷可用
- **现象**: 生成卷纲后，只有第一卷能生成章纲；"本卷章节生成"按钮对第2卷+无效
- **根因1**: `outline_service.py:211` `vol_num = vol.get("volume_number", 1)` — LLM返回的JSON中若无`volume_number`字段，所有卷都存为volume_number=1
- **根因2**: `upsert_chapter_outline` WHERE条件只用`chapter_number`，不含`volume_number`，导致不同卷同章号互相覆盖
- **验收**: 生成3卷后，每卷都能独立生成章纲，章纲按卷正确归属

### Bug 3: 总纲需人工输入
- **现象**: 大纲编辑器中总纲只能手动填写，没有AI自动生成入口
- **根因**: `OutlineEditor.vue` 缺少"AI生成总纲"按钮；后端已有`generate_master_from_conversation`但需要conv_id，需要新增一个基于小说设定直接生成的接口
- **验收**: 大纲编辑器有"AI生成"按钮，点击后基于小说idea/世界观/人物自动填充总纲字段

### Bug 4: 故事线/人物弧光AI生成没用
- **现象**: StorylineManager中AI生成按钮点击后无反馈，生成结果不显示
- **根因1**: `aiGenerateStorylines/Arcs/Scenes` 函数没有错误处理，失败时静默
- **根因2**: `generate_arcs_ai` 中 `any(c["id"] == char_id for c in characters)` — LLM生成的`character_id`是假设值(1,2,3...)，不匹配DB中实际ID，导致所有弧光被过滤掉
- **补充需求**: AI生成结果应先预览，允许用户修改后再确认保存
- **验收**: AI生成有加载状态和错误提示；人物弧光AI生成能正确关联实际人物；生成结果可预览修改后确认

---

## 验收标准

1. ✅ 概览tab可编辑创意和标题，PUT请求成功
2. ✅ 多卷场景下每卷章纲独立生成和显示
3. ✅ 大纲编辑器有AI生成总纲功能，无需对话即可触发
4. ✅ 故事线/人物弧光/场景AI生成有反馈，人物弧光正确关联实际人物ID
5. ✅ AI生成结果支持预览和修改后确认

- Bug 3新增后端接口，需确认与现有outline_service集成正确
