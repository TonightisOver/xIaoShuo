## 修复的问题

### 1. ✅ 输入验证缺失

**问题描述**: 所有节点缺少输入验证，存在 prompt injection 风险

**修复方案**:
- 创建 `src/core/validation.py` 验证工具模块
- 实现以下验证函数：
  - `validate_idea()`: 验证创意输入，检测可疑内容
  - `validate_novel_type()`: 验证小说类型白名单
  - `validate_target_words()`: 验证目标字数范围
  - `validate_json_string()`: 验证 JSON 字符串长度

**验证规则**:
- 创意最大长度: 1000 字符
- 移除控制字符
- 检测 prompt injection 模式（"ignore previous instructions"等）
- 小说类型白名单: 玄幻、仙侠、武侠、都市、科幻等 13 种
- 目标字数范围: 10,000 - 10,000,000

**修改的文件**:
- `src/core/validation.py` (新增)
- `src/core/langgraph/nodes/idea_expansion.py` (更新)

**测试覆盖**:
- 新增 13 个单元测试 (`tests/unit/test_core/test_validation.py`)
- 测试覆盖: 有效输入、空输入、过长输入、可疑内容、边界条件

### 2. ✅ JSON 解析错误处理不完善

**问题描述**: JSON 解析失败直接降级，可能丢失部分有效信息

**修复方案**:
- 创建 `src/core/json_utils.py` JSON 工具模块
- 实现智能 JSON 解析：
  - `safe_json_parse()`: 安全解析，支持部分提取和降级
  - `_extract_json_block()`: 从文本中提取 JSON 块（支持 ```json``` 代码块）
  - `_fix_common_json_issues()`: 修复常见问题（尾部逗号、BOM 等）
  - `validate_json_structure()`: 验证 JSON 结构完整性

**解析策略**:
1. 尝试直接解析
2. 提取 JSON 代码块
3. 修复常见格式问题
4. 验证必需字段
5. 降级到 fallback

**修改的文件**:
- `src/core/json_utils.py` (新增)
- `src/core/langgraph/nodes/world_building.py` (更新)
- `src/core/langgraph/nodes/character_design.py` (更新)
- `src/core/langgraph/nodes/outline_generation.py` (更新)

**测试覆盖**:
- 新增 9 个单元测试 (`tests/unit/test_core/test_json_utils.py`)
- 测试场景: 有效 JSON、代码块提取、尾部逗号修复、结构验证
