# 技术设计: 自定义文风（LLM 生成）

**变更 ID**: CHANGE-014  
**创建时间**: 2026-05-16

## 1. 设计概述

在 CHANGE-010 的基础上扩展，添加"自定义"风格选项。用户输入自然语言描述后，调用 LLM 生成具体的风格指令，保存到数据库，生成时注入到 prompt。

## 2. 数据流

```
用户输入描述 → POST /api/v1/style/generate → LLM 生成风格指令 → 返回给前端预览
                                                                    ↓
用户确认 → 保存到 novels.writing_style_prompt → 生成时注入 prompt
```

## 3. API 设计

### 新增端点

```
POST /api/v1/style/generate
Body: {"description": "江南的文风，带一点灰色幽默"}
Response: {"style_prompt": "请使用细腻优美的文风，句式长短交错..."}
```

### 修改端点

`POST /api/v1/projects` 和 `PUT /api/v1/projects/{id}`:
- 新增字段: `custom_style_description`, `writing_style_prompt`

## 4. 风格注入逻辑变更

当前逻辑（validation.py）:
```python
style_instruction = WRITING_STYLES.get(state.get("writing_style", ""), "")
```

新逻辑:
```python
if state.get("writing_style") == "自定义":
    style_instruction = state.get("writing_style_prompt", "")
else:
    style_instruction = WRITING_STYLES.get(state.get("writing_style", ""), "")
```

## 6. 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 风格生成时机 | 创建时即时生成 | 用户可预览确认 |
| 存储方式 | 两个字段分开存 | 原始描述可重新生成 |
| 注入方式 | 复用现有 style_instruction 变量 | 最小改动 |
