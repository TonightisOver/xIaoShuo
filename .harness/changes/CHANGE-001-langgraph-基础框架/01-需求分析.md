# 需求分析: LangGraph 基础框架

**变更 ID**: CHANGE-001  
**创建时间**: 2026-05-16  
**优先级**: P0

## 1. 需求背景

xIaoShuo 是一个 AI 驱动的中文网络小说生成平台，核心功能是通过 DeepSeek 大模型实现从创意到成品的全流程小说生成。LangGraph 基础框架是整个平台的核心，负责编排小说创作的多阶段流程。

**目标用户**: 开发团队（内部使用）  
**使用场景**: 所有小说生成功能都依赖此框架

## 2. 核心功能点

### 功能 1: NovelState 状态定义
**描述**: 定义小说创作过程中的完整状态结构  
**优先级**: P0  
**依赖**: 无

**状态字段**:
- 项目元信息：project_id, novel_type, target_words
- 创作内容：idea, world_setting, characters, relationships, outline, chapters
- 流程控制：current_stage, approval_status, revision_requests
- 质量指标：quality_scores
- 错误信息：errors

### 功能 2: StateGraph 图定义
**描述**: 创建 LangGraph StateGraph，定义小说创作流程  
**优先级**: P0  
**依赖**: 功能 1

**流程节点**:
1. 创意扩展（idea_expansion）
2. 世界观构建（world_building）
3. 人物设计（character_design）
4. 大纲生成（outline_generation）
5. 章节生成（chapter_generation）
6. 质量检查（quality_check）
7. 人工审核（human_review）

### 功能 3: 核心节点实现
**描述**: 实现各个节点的基础逻辑（暂时使用 mock 数据）  
**优先级**: P0  
**依赖**: 功能 2

**节点功能**:
- 每个节点接收 NovelState，返回更新后的 NovelState
- 暂时使用 mock 数据模拟 AI 生成结果
- 添加基础的错误处理

### 功能 4: Checkpointer 配置
**描述**: 配置 Checkpointer 支持中断恢复  
**优先级**: P1  
**依赖**: 功能 2

**实现方式**:
- 使用 MemorySaver（开发环境）
- 预留 SqliteSaver 接口（生产环境）

### 功能 5: 条件路由
**描述**: 实现质量检查后的条件路由  
**优先级**: P1  
**依赖**: 功能 3

**路由逻辑**:
- 质量分数 >= 0.8 → 下一阶段
- 质量分数 < 0.8 → 重新生成

### 功能 6: Human-in-the-Loop
**描述**: 实现人工审核节点，支持中断和恢复  
**优先级**: P1  
**依赖**: 功能 4

**实现方式**:
- 在关键节点（如大纲生成后）添加人工审核
- 支持审核通过/拒绝/修改
