# 技术设计: LangGraph 基础框架

**变更 ID**: CHANGE-001  
**创建时间**: 2026-05-16

## 1. 设计概述

使用 LangGraph StateGraph 实现小说创作流程编排，定义 NovelState 状态结构，实现 7 个核心节点，配置 Checkpointer 支持中断恢复。

## 2. 系统架构

### 2.1 架构图

```
用户请求
    ↓
创意扩展节点 (idea_expansion)
    ↓
世界观构建节点 (world_building)
    ↓
人物设计节点 (character_design)
    ↓
大纲生成节点 (outline_generation)
    ↓
章节生成节点 (chapter_generation)
    ↓
质量检查节点 (quality_check)
    ↓ (条件路由)
质量分数 >= 0.8 → 人工审核节点 (human_review) → END
质量分数 < 0.8 → 重新生成 (回到 chapter_generation)
```

### 2.2 核心组件

#### 组件 1: NovelState (状态定义)
**职责**: 定义小说创作过程中的完整状态  
**技术**: TypedDict  
**文件**: `src/core/langgraph/state.py`

#### 组件 2: StateGraph (流程图)
**职责**: 定义节点和边，编排创作流程  
**技术**: LangGraph StateGraph  
**文件**: `src/core/langgraph/graph.py`

#### 组件 3: 节点函数
**职责**: 实现各个阶段的处理逻辑  
**技术**: Python 函数  
**文件**: `src/core/langgraph/nodes/*.py`

#### 组件 4: Checkpointer
**职责**: 保存和恢复状态，支持中断恢复  
**技术**: MemorySaver / SqliteSaver  
**文件**: `src/core/langgraph/checkpointer.py`

### 2.3 LangGraph 设计

#### State 定义
```python
from typing import TypedDict, List, Dict, Optional

class NovelState(TypedDict):
    # 项目元信息
    project_id: str
    novel_type: str
    target_words: int
    
    # 创作内容
    idea: str
    world_setting: Optional[Dict]
    characters: List[Dict]
    relationships: Dict
    outline: Optional[Dict]
    chapter_outlines: List[Dict]
    chapters: List[Dict]
    
    # 流程控制
    current_stage: str
    approval_status: str  # pending/approved/rejected
    revision_requests: List[str]
    
    # 质量指标
    quality_scores: Dict[str, float]
    
    # 错误信息
    errors: List[str]
```

#### 节点定义
1. **idea_expansion**: 创意扩展 - 将用户的简单创意扩展为详细的故事概念
2. **world_building**: 世界观构建 - 构建小说的世界观、背景设定
3. **character_design**: 人物设计 - 设计主要人物和人物关系
4. **outline_generation**: 大纲生成 - 生成总体大纲和章节大纲
5. **chapter_generation**: 章节生成 - 生成具体章节内容
6. **quality_check**: 质量检查 - 检查生成内容的质量
7. **human_review**: 人工审核 - 等待人工审核和反馈

#### 边和路由
- idea_expansion → world_building（无条件）
- world_building → character_design（无条件）
- character_design → outline_generation（无条件）
- outline_generation → chapter_generation（无条件）
- chapter_generation → quality_check（无条件）
- quality_check → human_review（条件：quality_scores["overall"] >= 0.8）
- quality_check → chapter_generation（条件：quality_scores["overall"] < 0.8）
- human_review → END（无条件）

### 6.2 graph.py

```python
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver

from src.core.langgraph.state import NovelState
from src.core.langgraph.nodes import (
    idea_expansion,
    world_building,
    character_design,
    outline_generation,
    chapter_generation,
    quality_check,
    human_review
)

def create_novel_graph() -> StateGraph:
    """创建小说创作流程图"""
    # 创建图
    graph = StateGraph(NovelState)
    
    # 添加节点
    graph.add_node("idea_expansion", idea_expansion.node)
    graph.add_node("world_building", world_building.node)
    graph.add_node("character_design", character_design.node)
    graph.add_node("outline_generation", outline_generation.node)
    graph.add_node("chapter_generation", chapter_generation.node)
    graph.add_node("quality_check", quality_check.node)
    graph.add_node("human_review", human_review.node)
    
    # 设置入口点
    graph.set_entry_point("idea_expansion")
    
    # 添加边
    graph.add_edge("idea_expansion", "world_building")
    graph.add_edge("world_building", "character_design")
    graph.add_edge("character_design", "outline_generation")
    graph.add_edge("outline_generation", "chapter_generation")
    graph.add_edge("chapter_generation", "quality_check")
    
    # 添加条件路由
    graph.add_conditional_edges(
        "quality_check",
        should_continue,
        {
            "human_review": "human_review",
            "regenerate": "chapter_generation"
        }
    )
    
    graph.add_edge("human_review", END)
    
    # 配置 checkpointer
    checkpointer = MemorySaver()
    
    return graph.compile(checkpointer=checkpointer)

def should_continue(state: NovelState) -> str:
    """条件路由：根据质量分数决定下一步"""
    quality_score = state.get("quality_scores", {}).get("overall", 0)
    if quality_score >= 0.8:
        return "human_review"
    else:
        return "regenerate"
```

## 7. 性能和可扩展性

### 7.1 性能指标

- 单个节点执行时间: < 5 秒（mock 数据）
- 完整流程执行时间: < 30 秒（mock 数据）
- 状态保存时间: < 100ms

### 7.2 性能优化

- 使用异步节点（后续优化）
- 使用缓存减少重复计算
- 批量处理章节生成

### 7.3 扩展性设计

- 节点可独立扩展和替换
- 支持添加新节点
- 支持自定义路由逻辑
- 支持多种 Checkpointer

## 8. 安全性设计

- 状态数据不包含敏感信息
- Checkpointer 数据加密（生产环境）
- 输入验证（在 API 层）

## 9. 风险和挑战

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 状态设计不合理 | 高 | 充分参考需求，预留扩展字段 |
| Mock 数据不够真实 | 低 | 后续集成真实 AI 时优化 |
| Checkpointer 性能 | 中 | 开发环境用 MemorySaver，生产环境用 SqliteSaver |
