## 3. 核心实现

### 模块 1: NovelState 状态定义
**文件**: `src/core/langgraph/state.py`  
**功能**: 定义小说创作过程中的完整状态结构  
**关键代码**:
```python
class NovelState(TypedDict):
    # 15 个字段，包括项目元信息、创作内容、流程控制、质量指标、错误信息
```

### 模块 2: StateGraph 图定义
**文件**: `src/core/langgraph/graph.py`  
**功能**: 创建小说创作流程图，连接所有节点  
**关键代码**:
```python
def create_novel_graph() -> StateGraph:
    # 创建图、添加 7 个节点、连接边、配置条件路由、配置 Checkpointer
```

### 模块 3: 7 个核心节点
**文件**: `src/core/langgraph/nodes/*.py`  
**功能**: 实现各个阶段的处理逻辑（使用 mock 数据）  
**节点列表**:
1. idea_expansion - 创意扩展
2. world_building - 世界观构建
3. character_design - 人物设计
4. outline_generation - 大纲生成
5. chapter_generation - 章节生成
6. quality_check - 质量检查
7. human_review - 人工审核

### 模块 4: Checkpointer 配置
**文件**: `src/core/langgraph/checkpointer.py`  
**功能**: 根据环境选择合适的 Checkpointer  
**实现**: 开发环境用 MemorySaver，生产环境用 SqliteSaver

## 4. 技术要点

- **TypedDict 状态定义**: 使用 TypedDict 确保类型安全
- **不可变状态更新**: 所有节点返回新状态，不直接修改
- **条件路由**: 根据质量分数决定是否重新生成
- **Mock 数据**: 暂时使用 mock 数据模拟 AI 生成结果
- **完整类型注解**: 所有函数都有完整的类型注解

## 5. 遇到的问题和解决方案

### 问题 1: 如何设计状态结构
**解决方案**: 参考需求分析文档，定义了 15 个字段，涵盖项目元信息、创作内容、流程控制等

### 问题 2: 如何实现条件路由
**解决方案**: 使用 `add_conditional_edges` 和 `should_continue` 函数，根据质量分数路由

### 问题 3: Mock 数据如何设计
**解决方案**: 设计了真实的小说内容作为 mock 数据，便于后续测试
