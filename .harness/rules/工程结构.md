## 6. 文件命名规范

### 6.1 Python 文件
- **模块文件**: 小写 + 下划线（`project_service.py`）
- **类文件**: 小写 + 下划线，类名使用 PascalCase（`class ProjectService`）
- **测试文件**: `test_` 前缀（`test_project_service.py`）

### 6.2 目录
- **小写 + 下划线**（`langgraph/`, `test_services/`）
- **避免使用中文目录名**（除 `.harness/` 下的文档）

### 6.3 配置文件
- **小写 + 连字符**（`.env.example`, `docker-compose.yml`）
- **Python 配置**: `pyproject.toml`

## 7. 模块划分原则

### 7.1 单一职责
- 每个模块只负责一个明确的功能
- 避免"上帝类"（God Class）

### 7.2 分层清晰
```
API 层 (api/)
    ↓ 调用
服务层 (core/services/)
    ↓ 调用
数据层 (core/models/ + db/)
```

### 7.3 依赖方向
- **向下依赖**: API → Service → Model
- **禁止反向依赖**: Model 不能依赖 Service，Service 不能依赖 API

### 7.4 LangGraph 独立性
- LangGraph 流程定义在 `core/langgraph/` 下
- 节点函数可以调用 Service 层
- 但 Service 层不应直接依赖 LangGraph

## 8. 依赖关系约束

### 8.1 允许的依赖

```
api/routes/*.py
    → api/schemas/*.py
    → core/services/*.py
    → api/deps.py

core/services/*.py
    → core/models/*.py
    → core/langgraph/*.py
    → db/session.py
    → utils/*.py

core/langgraph/nodes/*.py
    → core/services/*.py
    → core/langgraph/state.py

core/models/*.py
    → db/base.py
```

### 8.2 禁止的依赖

```
❌ core/models/*.py → core/services/*.py
❌ core/services/*.py → api/routes/*.py
❌ db/*.py → core/models/*.py（除了 base.py）
❌ utils/*.py → core/services/*.py
```

## 9. 导入规范

### 9.1 导入顺序
```python
# 1. 标准库
import os
from typing import List, Dict

# 2. 第三方库
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session

# 3. 本地模块
from src.core.models.project import Project
from src.core.services.project_service import ProjectService
from src.api.deps import get_db
```

### 9.2 绝对导入 vs 相对导入
- **优先使用绝对导入**（`from src.core.models import Project`）
- **相对导入仅用于同一包内**（`from .state import NovelState`）

## 16. 正确示例

### ✅ 正确示例 1: 分层清晰
```python
# src/api/routes/projects.py
from fastapi import APIRouter, Depends
from src.core.services.project_service import ProjectService
from src.api.deps import get_db

router = APIRouter()

@router.post("/projects")
def create_project(data: dict, db: Session = Depends(get_db)):
    service = ProjectService(db)
    return service.create(data)
```

### ✅ 正确示例 2: 服务层调用模型层
```python
# src/core/services/project_service.py
from sqlalchemy.orm import Session
from src.core.models.project import Project

class ProjectService:
    def __init__(self, db: Session):
        self.db = db
    
    def create(self, data: dict) -> Project:
        project = Project(**data)
        self.db.add(project)
        self.db.commit()
        return project
```
