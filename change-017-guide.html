CHANGE-017: 故事线 / 人物弧光 / 场景管理

功能概述

为小说项目添加结构化的叙事管理能力：

功能说明用途
故事线贯穿多章的情节线索（主线/副线/暗线）管理多条并行的情节脉络
人物弧光人物在故事中的内在变化轨迹追踪角色成长/堕落/转变
场景故事中的重要时空场景管理地点和事件发生地
三元关联故事线↔人物↔场景的关系为图谱可视化提供数据

使用方式

方式一：前端界面

1. 打开小说详情页 → 点击"故事线" Tab

2. 点击"管理故事线"进入管理页面

3. 在管理页面可以：

  • 添加/删除故事线（主线/副线/暗线）
  • 添加/删除人物弧光（需要人物 ID）
  • 添加/删除场景

方式二：API 调用

故事线 CRUD

# 创建故事线
curl -X POST http://localhost/api/v1/projects/{novel_id}/storylines \
  -H "Content-Type: application/json" \
  -d '{"name": "主线：修仙之路", "type": "main", "description": "主角从凡人到仙人的成长", "key_events": [{"chapter": 1, "event": "觉醒灵根"}]}'

# 列出故事线
curl http://localhost/api/v1/projects/{novel_id}/storylines

# 更新故事线
curl -X PUT http://localhost/api/v1/projects/{novel_id}/storylines/{id} \
  -H "Content-Type: application/json" \
  -d '{"name": "主线：修仙之路", "type": "main", "status": "active"}'

# 删除故事线
curl -X DELETE http://localhost/api/v1/projects/{novel_id}/storylines/{id}

# 关联人物到故事线
curl -X POST http://localhost/api/v1/projects/{novel_id}/storylines/{id}/characters \
  -H "Content-Type: application/json" \
  -d '{"character_id": 1, "role_in_line": "主角"}'

人物弧光

# 创建人物弧光
curl -X POST http://localhost/api/v1/projects/{novel_id}/character-arcs \
  -H "Content-Type: application/json" \
  -d '{
    "character_id": 1,
    "arc_type": "growth",
    "description": "从懦弱少年成长为担当之人",
    "stages": [
      {"chapter_range": [1, 5], "state": "懦弱", "trigger": "家乡被毁"},
      {"chapter_range": [6, 15], "state": "独立", "trigger": "师父离去"},
      {"chapter_range": [16, 30], "state": "成熟", "trigger": "保护同伴"}
    ]
  }'

# 列出弧光
curl http://localhost/api/v1/projects/{novel_id}/character-arcs

场景

# 创建场景
curl -X POST http://localhost/api/v1/projects/{novel_id}/scenes \
  -H "Content-Type: application/json" \
  -d '{
    "name": "青云宗大殿",
    "location": "青云山顶",
    "description": "宗门议事之地",
    "appearances": [{"chapter": 3, "event": "入门仪式"}, {"chapter": 20, "event": "宗门大比"}]
  }'

# 列出场景
curl http://localhost/api/v1/projects/{novel_id}/scenes

获取三元关系（供图谱使用）

# 获取完整关系数据
curl http://localhost/api/v1/projects/{novel_id}/relations

# 返回格式：
{
  "storylines": [...],
  "character_arcs": [...],
  "scenes": [...],
  "storyline_character_links": [
    {"storyline_id": 1, "character_id": 1, "role_in_line": "主角"}
  ]
}

数据模型

故事线类型

type说明
main主线 - 核心情节
sub副线 - 辅助情节
hidden暗线 - 隐藏伏笔

弧光类型

arc_type说明示例
growth成长型从弱到强
fall堕落型从善到恶
flat平坦型始终如一
transformation转变型从敌到友

故事线状态

status说明
active进行中
resolved已解决/完结
abandoned已放弃

典型工作流

1. 创建故事线 → 定义主线和副线

2. 关联人物 → 每条线涉及哪些角色

3. 定义弧光 → 每个角色在故事中如何变化

4. 创建场景 → 重要地点和事件发生地

5. 查看关系 → GET /relations 获取完整图谱数据

6. 可视化 → CHANGE-018 将用树形图展示这些关系

提示：故事线的 key_events 和场景的 appearances 都引用了章节号，这样可以追踪"哪个事件发生在哪一章的哪个场景"。
注意：人物弧光需要先创建人物（在"人物"管理中），然后用人物 ID 创建弧光。
