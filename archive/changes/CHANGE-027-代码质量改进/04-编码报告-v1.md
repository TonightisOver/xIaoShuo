# CHANGE-027 编码报告 v1

## 执行摘要

按技术设计完成 6 个模块的编码实现（模块 D 经核查无需修改）。

## 模块实现详情

### 模块 A — 重试逻辑 ✅

**文件**: `src/core/llm/client.py`

- 移除 `@retry` 装饰器，改用 `AsyncRetrying` 上下文管理器
- `stop=stop_after_attempt(self.max_retries)` 运行时读取配置
- `wait=wait_exponential(multiplier=1, min=2, max=30)`
- 新增 `httpx.HTTPStatusError` 捕获：429/5xx 重试，401 直接抛出不重试
- 超出重试次数时捕获 `RetryError` 并记录结构化日志

### 模块 B — CORS 配置 ✅

**文件**: `src/core/config.py`, `src/api/main.py`

- `config.py` 新增 `CORS_ORIGINS: list[str] = ["http://localhost:5173", "http://localhost:3000"]`
- `main.py` 将 `allow_origins=["*"]` 改为 `allow_origins=settings.CORS_ORIGINS`

### 模块 C — 数据库索引 ✅

**文件**: `src/api/models/db_models.py`, `alembic/versions/20260519_add_status_indexes.py`

为以下 5 个 `status` 字段添加 `index=True`（`Novel.status` 和 `Task.status` 已有索引，无需修改）：
- `Volume.status`
- `Chapter.status`
- `Conversation.status`
- `Outline.status`
- `Storyline.status`

新建 alembic migration，down_revision 指向 `b0018a838516`（最新 migration）。

### 模块 D — 内存泄漏修复 ✅（无需修改）

经核查，`generate_novel_background()` 和 `_run_langgraph_pipeline()` 均已正确实现 `try/finally` 包裹 `unregister_progress_callback`。无需修改。

### 模块 E — 结构化日志 ✅

**文件**: `src/core/config.py`, `src/core/logging_config.py`, `src/core/llm/client.py`, `src/api/services/novel_generator.py`, `src/api/main.py`, `pyproject.toml`

- `config.py` 新增 `LOG_FORMAT: str = "console"`
- `logging_config.py` 重写为 structlog，支持 `"console"` 和 `"json"` 两种渲染器
- `setup_logging()` 新增 `log_format` 参数
- `client.py` 和 `novel_generator.py` 改用 `structlog.get_logger()`，关键日志改为结构化字段
- `main.py` 调用 `setup_logging` 时传入 `log_format=settings.LOG_FORMAT`
- `pyproject.toml` 添加 `structlog = ">=24.0.0"`

### 模块 F — response_model ✅

**文件**: `src/api/models/responses.py`, `src/api/routes/projects.py`

`responses.py` 新增 7 个响应模型：
- `NovelSummaryResponse` — 列表项
- `NovelDetailResponse` — 详情（继承 Summary）
- `NovelListResponse` — 列表+分页
- `CreateResponse` — 创建响应
- `StatusResponse` — 通用状态响应
- `ChapterResponse` — 章节响应
- `VolumeResponse` — 卷响应

`projects.py` 为 9 个核心路由添加 `response_model`：
- `POST /` → `CreateResponse`
- `GET /` → `NovelListResponse`
- `GET /{novel_id}` → `NovelDetailResponse`
- `PUT /{novel_id}` → `StatusResponse`
- `DELETE /{novel_id}` → `StatusResponse`
- `GET /{novel_id}/volumes/{volume_number}` → `VolumeResponse`
- `PUT /{novel_id}/volumes/{volume_number}` → `StatusResponse`
- `GET /{novel_id}/chapters/{chapter_number}` → `ChapterResponse`
- `PUT /{novel_id}/chapters/{chapter_number}` → `StatusResponse`

### 模块 G — N+1 注释 ✅

**文件**: `src/api/services/novel_manager.py`

在 `list_novels` 方法添加注释，说明无 N+1 问题。
