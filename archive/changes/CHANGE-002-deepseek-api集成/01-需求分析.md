# 需求分析: DeepSeek API 集成

**变更 ID**: CHANGE-002  
**创建时间**: 2026-05-16

## 1. 需求背景

当前 LangGraph 基础框架使用 mock 数据模拟 AI 生成，需要集成 DeepSeek API 实现真实的小说内容生成。

## 2. 业务需求

### 2.1 核心需求

**FR-001: API 连接配置**
- 配置 DeepSeek API 连接
- 管理 API Key（环境变量）
- 支持自定义 base_url

**FR-002: 节点 AI 生成**
- 创意扩展节点：调用 API 扩展用户创意
- 世界观构建节点：调用 API 生成世界观设定
- 人物设计节点：调用 API 设计人物和关系
- 大纲生成节点：调用 API 生成总体大纲和章节大纲
- 章节生成节点：调用 API 生成章节内容

**FR-003: Prompt 工程**
- 为每个节点设计专业的 prompt 模板
- 确保生成内容符合中文小说风格
- 控制生成内容的长度和质量

**FR-004: 错误处理**
- API 调用失败时的重试机制
- 超时处理
- 错误信息记录到 state["errors"]

### 2.2 非功能需求

**NFR-001: 性能**
- 单个节点 API 调用时间 < 10s
- 完整流程执行时间 < 60s
- 支持异步调用

**NFR-002: 可靠性**
- API 调用失败自动重试（最多 3 次）
- 超时时间设置为 30s
- 记录详细的错误日志

**NFR-003: 安全性**
- API Key 通过环境变量管理
- 不在代码中硬编码 API Key
- 不在日志中输出 API Key

**NFR-004: 可维护性**
- 统一的 LLM 客户端封装
- 可配置的模型参数（temperature, max_tokens）
- 易于切换到其他 LLM 提供商

## 4. 功能点分解

### 4.1 配置管理

**模块**: `src/core/config.py`

**功能**:
- 加载环境变量
- 提供配置类（Settings）
- 验证必需的配置项

**配置项**:
- `DEEPSEEK_API_KEY` - API 密钥
- `DEEPSEEK_BASE_URL` - API 端点（默认 https://api.deepseek.com/v1）
- `DEEPSEEK_MODEL` - 模型名称（默认 deepseek-chat）
- `DEEPSEEK_TEMPERATURE` - 温度参数（默认 0.7）
- `DEEPSEEK_MAX_TOKENS` - 最大 token 数（默认 2000）
- `DEEPSEEK_TIMEOUT` - 超时时间（默认 30s）

### 4.2 LLM 客户端

**模块**: `src/core/llm/client.py`

**功能**:
- 创建 ChatOpenAI 实例
- 封装 API 调用
- 实现重试机制
- 错误处理

**接口**:
```python
async def generate_text(prompt: str, max_tokens: int = 2000) -> str:
    """生成文本"""
    pass

async def generate_json(prompt: str, schema: dict) -> dict:
    """生成 JSON 格式的内容"""
    pass
```

### 4.3 Prompt 模板

**模块**: `src/core/llm/prompts.py`

**功能**:
- 定义各个节点的 prompt 模板
- 使用 LangChain PromptTemplate

**模板列表**:
1. `IDEA_EXPANSION_PROMPT` - 创意扩展
2. `WORLD_BUILDING_PROMPT` - 世界观构建
3. `CHARACTER_DESIGN_PROMPT` - 人物设计
4. `OUTLINE_GENERATION_PROMPT` - 大纲生成
5. `CHAPTER_GENERATION_PROMPT` - 章节生成

### 4.4 节点改造

**改造内容**:
- 将节点函数改为异步函数（async def）
- 调用 LLM 客户端生成内容
- 保留 mock 数据作为降级方案
- 添加错误处理

**改造节点**:
- `idea_expansion.py`
- `world_building.py`
- `character_design.py`
- `outline_generation.py`
- `chapter_generation.py`

**不改造节点**:
- `quality_check.py` - 使用规则评分，不需要 API
- `human_review.py` - 等待人工审核，不需要 API

## 5. 数据流

### 5.1 API 调用流程

```
用户输入
  ↓
节点函数（异步）
  ↓
LLM 客户端
  ↓
构建 Prompt
  ↓
调用 DeepSeek API
  ↓
解析响应
  ↓
更新状态
  ↓
返回新状态
```

### 5.2 错误处理流程

```
API 调用
  ↓
失败？
  ├─ 是 → 重试（最多 3 次）
  │         ↓
  │       仍失败？
  │         ├─ 是 → 记录错误 → 使用降级方案（mock 数据）
  │         └─ 否 → 返回结果
  └─ 否 → 返回结果
```
