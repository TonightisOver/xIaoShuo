# 需求分析: 人机交互升级

**变更 ID**: CHANGE-020  
**创建时间**: 2026-05-17  
**优先级**: P0

## 1. 需求背景

当前人机交互（CHANGE-013）只是"讨论+建议"模式，结论需要手动应用。用户希望对话能直接"确定"设定并驱动后续生成流程，形成完整闭环：

```
讨论 → 确认设定 → 生成总纲 → 生成卷纲 → 生成章节
```

## 2. 核心功能点

### 功能 1: 对话中确定设定
**描述**: 对话过程中，用户可以将某条 AI 回复标记为"确定"，直接写入对应设定  
**交互**:
- AI 回复中如果包含结构化建议，显示"确定此设定"按钮
- 点击后直接写入世界观/人物/故事线（不是"建议"，是"确定"）
- 确定后该条消息标记为"已确定"状态

### 功能 2: 对话驱动大纲生成
**描述**: 对话结束后，可一键将讨论结果生成为总纲，并自动触发卷纲生成  
**流程**:
1. 对话中讨论情节走向
2. 用户点击"确定为总纲"
3. LLM 整理对话内容为结构化总纲
4. 自动保存到 outlines 表
5. 提示用户"是否继续生成卷纲？"

### 功能 3: 一键流程（对话→设定→大纲→生成）
**描述**: 从对话开始，一步步引导完成整个创作准备  
**流程**:
```
[创作对话] → 确定世界观 → 确定人物 → 确定故事线 → 生成总纲 → 生成卷纲 → 开始生成
```

### 功能 4: 消息确认状态
**描述**: messages 表添加 confirmed 字段，标记哪些消息已被确认为设定  
**数据变更**: messages 表添加 `confirmed_as VARCHAR(50)` 字段（null/world/character/storyline/outline）

## 3. API 设计

```
# 确认消息为设定
POST /api/v1/projects/{id}/conversations/{conv_id}/messages/{msg_id}/confirm
Body: {"confirm_as": "world|character|storyline|outline"}

# 从对话生成总纲并自动生成卷纲
POST /api/v1/projects/{id}/conversations/{conv_id}/generate-outline
Response: {"master_outline": {...}, "volumes_generated": true}
```

## 4. 前端交互设计

对话页面中每条 AI 回复：
```
┌─────────────────────────────────────┐
│ AI: 建议世界观设定如下：            │
│ 这是一个以灵气为基础的修仙世界...   │
│                                     │
│ [确定为世界观] [确定为人物] [忽略]   │
└─────────────────────────────────────┘
```

对话结束后：
```
┌─────────────────────────────────────┐
│ 对话已结束                          │
│                                     │
│ [应用到设定] [生成故事线] [生成总纲] │
│                                     │
│ [一键生成：总纲→卷纲→开始写作]      │
└─────────────────────────────────────┘
```

## 5. 验收标准

- [ ] AI 回复可标记为"确定"并直接写入设定
- [ ] 确定后消息显示"已确定为XX"标记
- [ ] 对话结束后可一键生成总纲
- [ ] 生成总纲后可自动生成卷纲
- [ ] "一键流程"按钮可触发完整链路
- [ ] 所有测试通过
