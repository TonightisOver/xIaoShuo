# 需求分析: 按卷生成

**变更 ID**: CHANGE-011  
**创建时间**: 2026-05-16  
**优先级**: P1

## 1. 需求背景

当前系统只支持"一次性全量生成"——从创意到所有章节一口气跑完。对于长篇小说（10万字+），这种方式存在问题：
- 生成时间过长（可能数十分钟）
- 无法在中间调整方向
- 前面的卷生成质量无法影响后续卷的生成

用户希望支持"按卷生成"模式：先生成总纲，再分卷生成大纲，最后逐卷生成章节内容。每卷生成前可以审核和调整。

**目标用户**: 长篇小说创作者  
**使用场景**: 10万字以上的长篇小说，需要分阶段控制质量

## 2. 核心功能点

### 功能 1: 卷结构定义
**描述**: 在大纲生成阶段，将章节按卷划分  
**优先级**: P0

**数据结构**:
```
总纲 (outline)
├── 卷一大纲 (volume 1)
│   ├── 第1章大纲
│   ├── 第2章大纲
│   └── ...
├── 卷二大纲 (volume 2)
│   ├── 第N章大纲
│   └── ...
└── 卷三大纲 (volume 3)
    └── ...
```

### 功能 2: 按卷生成触发
**描述**: 用户可以选择生成指定卷的章节内容  
**优先级**: P0  
**API**: `POST /api/v1/projects/{novel_id}/generate-volume`  
**参数**: `volume_number` (int)

### 功能 3: 卷大纲基于总纲分析生成
**描述**: 每卷的详细大纲根据总纲进行分析后生成，确保前后连贯  
**优先级**: P0  
**实现**: 新增 volume_outline_generation 节点

### 功能 4: 前端卷管理 UI
**描述**: 小说详情页展示卷结构，每卷可独立触发生成  
**优先级**: P1

## 3. 数据模型变更

### 新增表: volumes

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | 关联小说 |
| volume_number | INTEGER | 卷号 |
| title | VARCHAR(200) | 卷标题 |
| summary | TEXT | 卷概要 |
| outline | JSON | 卷详细大纲 |
| status | VARCHAR(20) | draft/generating/completed |
| chapter_start | INTEGER | 起始章节号 |
| chapter_end | INTEGER | 结束章节号 |

### 修改: chapters 表

添加 `volume_number INTEGER` 字段，关联到卷。

## 4. 工作流变更

**当前流程（一次性）**:
```
创意 → 世界观 → 人物 → 大纲(全部章节) → 章节(全部) → 质检
```

**新流程（按卷）**:
```
创意 → 世界观 → 人物 → 总纲(含卷划分)
                                ↓
                    [用户选择生成某卷]
                                ↓
                    卷大纲细化 → 该卷章节生成 → 质检
```

## 5. 非功能需求

- 兼容现有"一次性生成"模式（不选卷 = 全量生成）
- 已生成的卷内容作为上下文传入后续卷的生成
- 卷之间的情节连贯性由 prompt 保证

## 6. 验收标准

- [ ] 大纲生成后自动划分为多卷
- [ ] 可通过 API 触发单卷生成
- [ ] 单卷生成时，前面已完成卷的内容作为上下文
- [ ] 前端显示卷结构，每卷可独立操作
- [ ] volumes 表正确存储卷信息
- [ ] chapters 表关联 volume_number
- [ ] 所有现有测试通过 + 新增测试
