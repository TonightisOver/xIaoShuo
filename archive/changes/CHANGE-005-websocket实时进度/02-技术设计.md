## 架构方案

### 事件总线（Progress Event Bus）

- 文件：`src/api/services/progress_event_bus.py`
- 模式：asyncio.Queue pub/sub
- 每个订阅者持有独立 Queue，发布时广播到所有订阅者队列
- 按 task_id 隔离订阅关系

### WebSocket 端点

- 文件：`src/api/routes/ws.py`
- 路径：`/ws/tasks/{task_id}`
- 连接后自动订阅对应 task_id 的事件流
- 30 秒心跳 ping 保活
- 断开时自动取消订阅并清理队列

### 生成流程改造

- `src/api/services/novel_generator.py`：ainvoke 改为 astream，逐步产出中间结果
- `src/core/langgraph/nodes/chapter_generation.py`：注册回调，章节生成时发布 chapter_progress 事件

### 事件消息格式

```json
{
  "event": "chapter_progress",
  "task_id": "xxx",
  "data": { ... },
  "timestamp": "2026-05-16T10:00:00Z"
}
```
