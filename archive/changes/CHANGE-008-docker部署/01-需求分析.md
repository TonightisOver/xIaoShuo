# 需求分析

## 变更编号
CHANGE-008

## 变更标题
Docker 容器化部署

## 需求背景
项目需要标准化部署流程，实现一键部署到云服务器，降低运维复杂度。

## 功能需求

1. **容器化打包** — 前端(Node.js构建) + 后端(FastAPI) 合并为单一镜像
2. **服务编排** — PostgreSQL、API、Nginx 三服务通过 docker-compose 编排
3. **反向代理** — Nginx 代理 HTTP 请求及 WebSocket 连接
4. **一键部署** — 提供 deploy.sh 脚本，在 Ubuntu 22.04 上自动安装依赖并启动服务
5. **部署文档** — 提供 deploy-guide.html 可视化部署指南

## 非功能需求

- 镜像体积尽量小（multi-stage build, slim 基础镜像）
- 数据库数据持久化（Docker volume）
- 服务健康检查
- 环境变量模板化（.env.docker）

## 目标环境

| 项目 | 值 |
|------|------|
| 服务器 | 115.190.142.169 |
| 操作系统 | Ubuntu 22.04 |
| 对外端口 | 80 |

## 验收标准

- 3 个容器正常运行
- Health check 返回 200 OK
- 前端页面可通过 80 端口访问
- WebSocket 连接正常
