# 需求分析: Web API (FastAPI)

**变更 ID**: CHANGE-003  
**创建时间**: 2026-05-16  
**优先级**: P0

## 1. 需求背景

xIaoShuo 平台需要对外提供 HTTP API 接口，让客户端（前端/第三方）能够创建小说生成任务、查询任务状态、获取生成结果。

**目标用户**: 前端应用、API 调用方  
**使用场景**: 所有用户交互都通过 API 进行

## 2. 核心功能点

### 功能 1: 健康检查端点
**描述**: 提供服务健康状态查询  
**优先级**: P0  
**端点**: `GET /api/v1/health`

### 功能 2: 创建小说生成任务
**描述**: 接收创意、类型、字数参数，创建后台生成任务  
**优先级**: P0  
**端点**: `POST /api/v1/novels`  
**输入验证**: 创意 10-1000 字，类型必须在支持列表中

### 功能 3: 查询任务详情
**描述**: 根据 task_id 查询任务状态、进度、结果  
**优先级**: P0  
**端点**: `GET /api/v1/novels/{task_id}`

### 功能 4: 列出任务列表
**描述**: 分页列出任务，支持状态过滤  
**优先级**: P1  
**端点**: `GET /api/v1/novels?status=&limit=&offset=`

## 3. 非功能需求

- 异步处理：生成任务在后台执行，API 立即返回 202
- CORS：允许跨域访问
- 文档：自动生成 Swagger UI 和 ReDoc
- 输入验证：Pydantic 模型验证

## 4. 验收标准

- [ ] 4 个 API 端点可正常访问
- [ ] 创建任务返回 202 + task_id
- [ ] 无效输入返回 400/422
- [ ] 不存在的任务返回 404
- [ ] Swagger 文档可访问
- [ ] 9 个测试用例通过
