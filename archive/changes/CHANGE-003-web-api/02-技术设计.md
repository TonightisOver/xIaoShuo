# 技术设计: Web API (FastAPI)

**变更 ID**: CHANGE-003  
**创建时间**: 2026-05-16

## 1. 设计概述

使用 FastAPI 框架构建异步 RESTful API，采用分层架构（routes → services → models），通过 BackgroundTasks 实现异步任务执行。

## 2. 系统架构

```
HTTP 请求
    ↓
FastAPI (CORS + 验证)
    ↓
Routes (novels.py / health.py)
    ↓
Services (task_manager.py / novel_generator.py)
    ↓
LangGraph 工作流
```

### 2.1 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| FastAPI App | `src/api/main.py` | 应用入口，中间件，路由注册 |
| Novels Router | `src/api/routes/novels.py` | 任务 CRUD 端点 |
| Health Router | `src/api/routes/health.py` | 健康检查 |
| TaskManager | `src/api/services/task_manager.py` | 任务状态管理 |
| NovelGenerator | `src/api/services/novel_generator.py` | 后台生成执行 |
| Request Models | `src/api/models/requests.py` | Pydantic 请求模型 |
| Response Models | `src/api/models/responses.py` | Pydantic 响应模型 |
| Validation | `src/core/validation.py` | 输入验证逻辑 |

### 2.2 请求/响应设计

**CreateNovelRequest**:
- idea: str (10-1000 字符)
- novel_type: str (枚举)
- target_words: int (10000-10000000)

**TaskResponse** (202):
- task_id, status, created_at, estimated_duration_minutes

**TaskDetailResponse**:
- task_id, status, progress, timestamps, result, errors

### 2.3 异步执行模型

1. POST 请求 → 创建 Task 记录（status=pending）→ 返回 202
2. BackgroundTasks 触发 generate_novel_background()
3. 客户端轮询 GET 获取进度

## 3. 技术选型

| 决策 | 选择 | 理由 |
|------|------|------|
| 框架 | FastAPI | 异步原生，自动文档，Pydantic 集成 |
| 验证 | Pydantic v2 | 类型安全，自动 422 错误 |
| 异步任务 | BackgroundTasks | 轻量，无需 Celery |
| API 版本 | /api/v1/ 前缀 | 支持未来版本迭代 |
