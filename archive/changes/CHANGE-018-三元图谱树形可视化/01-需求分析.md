# 需求分析: 三元图谱树形可视化

**变更 ID**: CHANGE-018  
**创建时间**: 2026-05-17  
**优先级**: P1

## 1. 需求背景

CHANGE-017 建立了故事线/人物弧光/场景的数据模型和 API。现在需要将这些三元关系以树形图的方式可视化展示，让用户直观看到人物-情节-场景之间的关联。

## 2. 核心功能点

### 功能 1: 树形图谱展示
**描述**: 以树形结构展示小说的叙事关系  
**技术**: D3.js 树形布局  
**结构**:
```
小说项目（根节点）
├── 故事线
│   ├── [主线] 修仙之路
│   │   ├── 人物: 张三（主角）
│   │   ├── 人物: 李四（对手）
│   │   └── 关键事件: 觉醒灵根(Ch.1), 拜入宗门(Ch.5)
│   └── [副线] 感情线
│       └── 人物: 王五（恋人）
├── 人物弧光
│   ├── 张三: growth（懦弱→独立→成熟）
│   └── 李四: transformation（敌→友）
└── 场景
    ├── 青云宗大殿 (Ch.3, Ch.20)
    └── 魔域深渊 (Ch.15, Ch.25)
```

### 功能 2: 交互功能
**描述**: 节点可展开/折叠，悬停显示详情  
**优先级**: P1

### 功能 3: 从 API 加载数据
**描述**: 调用 GET /relations 获取数据，转换为 D3 树形结构  
**优先级**: P0

## 3. 技术选型

- **D3.js**: 树形布局（d3-hierarchy + d3-tree）
- **集成方式**: 作为 Vue 组件，通过 CDN 引入 D3 或 npm 安装
- **渲染**: SVG

## 4. 验收标准

- [ ] 树形图正确展示故事线/人物/场景关系
- [ ] 节点可展开/折叠
- [ ] 悬停显示详情
- [ ] 数据从 GET /relations API 加载
- [ ] 响应式布局（适应不同屏幕）
- [ ] 所有测试通过
