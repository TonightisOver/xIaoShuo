# 专家评审报告: 三元图谱树形可视化

**变更 ID**: CHANGE-018  
**版本**: v1  
**评审时间**: 2026-05-17

## 评审维度

### 1. 架构设计

| 评审项 | 评价 |
|--------|------|
| 技术选型 | D3.js 是树形可视化的成熟方案，社区活跃，文档完善 |
| 组件设计 | RelationGraph 独立页面，不侵入现有组件逻辑 |
| 数据流 | 复用 CHANGE-017 的 GET /relations API，无需新增后端接口 |
| 树形结构 | root → 分类（故事线/人物弧光/场景）→ 条目 → 详情，层级清晰 |
| 入口设计 | NovelDetail 故事线 Tab 内添加按钮，发现路径自然 |
| 渲染方式 | SVG 适合节点数有限的树形图，性能无忧 |

### 2. 安全性

| 评审项 | 评价 |
|--------|------|
| XSS 风险 | 节点文本通过 D3 text() 方法设置，自动转义 HTML |
| 数据来源 | 仅读取已有 API 数据，无新增写入操作 |
| 路由权限 | 与 NovelDetail 同级路由，继承现有访问控制 |

### 3. 性能

| 评审项 | 评价 |
|--------|------|
| Bundle 大小 | RelationGraph chunk ~50KB（含 D3 tree-shaking 后），可接受 |
| 渲染性能 | 单项目关系数据通常 < 200 节点，SVG 渲染无压力 |
| 懒加载 | 路由级代码分割，仅访问图谱页时加载 D3 |
| 内存占用 | D3 树形计算为一次性操作，无持续内存开销 |

### 4. 可维护性

| 评审项 | 评价 |
|--------|------|
| 代码可读性 | 数据转换与渲染逻辑分离，buildHierarchy 函数职责单一 |
| 扩展性 | 新增节点类型只需在 nodeColor 映射中添加颜色 |
| 测试友好 | 数据转换逻辑为纯函数，可独立测试 |

## 评审意见

### 建议（非阻塞）

1. **按需引入 D3**：仅引入 d3-hierarchy/d3-shape/d3-selection，可减少约 30KB bundle。
2. **缩放支持**：节点较多时可添加 d3.zoom 支持平移缩放。
3. **导出功能**：后续可添加 SVG 导出为 PNG 的功能，方便用户分享。

### 亮点

- 纯前端实现，零后端改动，复用现有 API
- 节点颜色编码直观区分实体类型
- 水平树布局适合中文长文本节点名称显示

## 结论

**通过**。技术选型合理，实现完整，性能无忧。纯前端变更风险低，不影响现有功能。建议项为后续增强方向，不阻塞合入。
