## 端到端流程验证

### 流程 1: 小说全生命周期

```
POST /projects → 201 (创建小说)
GET  /projects → 200 (列表包含新小说)
GET  /projects/{id} → 200 (详情正确)
PUT  /projects/{id} → 200 (更新标题)
DELETE /projects/{id} → 200 (删除成功)
GET  /projects/{id} → 404 (已删除)
```

### 流程 2: 子资源 CRUD

```
POST /projects/{id}/characters → 201
GET  /projects/{id}/characters → 200 (包含新角色)
PUT  /projects/{id}/characters/{c_id} → 200
DELETE /projects/{id}/characters/{c_id} → 200
GET  /projects/{id}/characters/{c_id} → 404
```

### 流程 3: 生成 + 自动持久化

```
POST /projects/{id}/generate → 202 (触发生成)
[等待生成完成]
GET  /projects/{id}/world → 200 (世界观已存储)
GET  /projects/{id}/characters → 200 (角色已存储)
GET  /projects/{id}/chapters → 200 (章节已存储)
```

### 流程 4: CASCADE 删除

```
POST /projects → 201
POST /projects/{id}/characters → 201 (添加角色)
DELETE /projects/{id} → 200
[验证 characters 表无孤立记录]
```

## 结论

所有端到端流程验证通过，子资源 CRUD 与级联删除行为符合预期。
