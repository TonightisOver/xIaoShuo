# 需求分析

## 变更编号
CHANGE-007

## 变更标题
小说项目管理

## 背景
当前系统使用扁平的 task 模型管理生成任务，无法有效组织小说的多维度数据（世界观、力量体系、角色、章节等）。需升级为"小说项目"模型，支持子资源的独立 CRUD。

## 需求目标
1. 引入 Novel 项目模型，作为顶层聚合根
2. 支持世界观设定（WorldSetting）、力量体系（PowerSystem）、角色（Character）、章节（Chapter）作为子资源
3. 提供完整的 RESTful API（20+ 端点）
4. 生成器完成后自动持久化结果到子表
5. 保持与现有 task 模型的兼容（task 关联 novel_id）

## 功能范围
| 模块 | 说明 |
|------|------|
| 数据模型 | 5 张新表 + FK + CASCADE 删除 |
| API 层 | /api/v1/projects CRUD + 子资源端点 |
| 服务层 | NovelManager 统一管理 |
| 生成器集成 | _persist_to_novel 自动写入 |
| 数据库迁移 | Alembic 迁移脚本 |

## 非功能需求
- 级联删除保证数据一致性
- 子资源查询需支持按项目过滤
- API 响应时间 < 200ms（单资源操作）

## 验收标准
- 13 项测试全部通过
- 所有 CRUD 端点可用
- 删除项目时子资源同步清除
