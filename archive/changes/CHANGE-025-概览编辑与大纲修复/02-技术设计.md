## 修复方案

### Fix 1: 概览创意编辑 (NovelDetail.vue)

在 overview tab 中添加编辑模式：
- 新增 `editingOverview` ref 和 `overviewForm` ref
- 点击"编辑"进入编辑模式，显示 input/textarea
- 点击"保存"调用 `PUT /api/v1/projects/{novel_id}`，更新 `title` 和 `idea`
- 保存成功后退出编辑模式，刷新 novel 数据

API: `PUT /api/v1/projects/{novel_id}` 已存在（需确认）

### Fix 2: 多卷章纲修复 (outline_service.py)

**问题1**: `generate_volume_outlines` 中 `vol_num = vol.get("volume_number", 1)` 默认值错误

修复：使用枚举索引作为fallback
```python
for idx, vol in enumerate(volumes, start=1):
    vol_num = vol.get("volume_number") or idx
```

**问题2**: `upsert_chapter_outline` WHERE条件缺少 `volume_number`

修复：在WHERE条件中加入 `Outline.volume_number == volume_number`
```python
select(Outline).where(and_(
    Outline.novel_id == novel_id,
    Outline.level == "chapter",
    Outline.volume_number == volume_number,  # 加入此条件
    Outline.chapter_number == chapter_number
))
```

### Fix 3: AI生成总纲 (OutlineEditor.vue + outlines.py)

**后端**: 在 `OutlineService` 新增 `generate_master_from_novel` 方法，基于小说设定（idea/world/characters）直接生成总纲，无需conv_id。

在 `outlines.py` 新增路由：
```
POST /api/v1/projects/{novel_id}/outlines/generate-master
```

**前端**: `OutlineEditor.vue` 总纲区域添加"AI生成"按钮，调用新接口，成功后填充表单字段。

### Fix 4: 故事线AI生成修复 (StorylineManager.vue + storyline_service.py)

**后端**: `generate_arcs_ai` 修复 — 不再过滤character_id，改为按顺序分配实际人物ID：
```python
# 将LLM返回的弧光按顺序映射到实际人物
for i, arc in enumerate(data):
    if i < len(characters):
        actual_char_id = characters[i]["id"]
        arc_id = await self.create_character_arc(novel_id, character_id=actual_char_id, ...)
```

**前端**: `StorylineManager.vue` 改造：
- AI生成结果先存入 `pendingXxx` ref，显示预览面板
- 用户可在预览中修改各项内容
- 点击"确认保存"才真正调用POST接口保存
- 添加错误提示
