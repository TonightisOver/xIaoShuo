# 04-编码报告-v1

**变更**: CHANGE-025  
**日期**: 2026-05-18

## 修改文件

### 后端

**`src/api/services/outline_service.py`**
- `generate_volume_outlines`: 改用 `enumerate(volumes, start=1)` 作为 `volume_number` fallback，修复LLM不返回该字段时所有卷存为1的bug
- `upsert_chapter_outline`: WHERE条件加入 `Outline.volume_number == volume_number`，防止不同卷同章号互相覆盖
- 新增 `generate_master_from_novel(novel_id)`: 基于小说idea/世界观/人物直接生成总纲，无需对话

**`src/api/routes/outlines.py`**
- 新增 `POST /{novel_id}/outlines/generate-master` 路由

**`src/api/services/storyline_service.py`**
- `generate_arcs_ai`: 改为按枚举索引映射实际人物ID，不再依赖LLM返回的假设ID

### 前端

**`frontend/src/views/NovelDetail.vue`**
- 概览tab添加编辑模式：编辑/保存/取消按钮，标题和创意可编辑，调用 `PUT /api/v1/projects/{novel_id}`

**`frontend/src/views/OutlineEditor.vue`**
- 总纲区域添加"AI生成"按钮，调用 `POST /outlines/generate-master`，成功后填充表单

**`frontend/src/views/StorylineManager.vue`**
- 完整重写：AI生成结果先存入pending状态显示预览面板，用户可修改后点击"确认保存"调用PUT更新
- 添加错误提示banner
- 加载人物列表用于弧光编辑中的人物选择器
- 人物弧光显示实际人物名称而非ID

### 测试

**`tests/unit/test_change025_fixes.py`**
- 5个单元测试覆盖volume_number fallback和arc character_id映射逻辑
