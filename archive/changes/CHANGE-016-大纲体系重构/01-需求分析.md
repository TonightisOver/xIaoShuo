# 需求分析: 大纲体系重构

**变更 ID**: CHANGE-016  
**创建时间**: 2026-05-16  
**优先级**: P0

## 1. 需求背景

当前大纲系统是扁平的：一个 outline JSON + 一个 chapter_outlines 数组。无法表达层级关系，也无法支持"人机交互讨论后逐步细化"的创作流程。

需要重构为三级大纲体系：
- **总纲**：故事的核心脉络（开端/发展/高潮/结局），通过人机交互确定
- **卷纲**：每卷的详细情节线，基于总纲拆分
- **章纲**：每章的具体场景和情节点

## 2. 核心功能点

### 功能 1: 总纲管理
**描述**: 总纲是小说的顶层设计，包含核心冲突、主线剧情、结局走向  
**优先级**: P0  
**来源**: 人机交互对话确定 → 或手动编辑  
**数据结构**:
```json
{
  "premise": "核心前提/设定",
  "main_conflict": "主要冲突",
  "plot_arcs": [
    {"name": "主线", "description": "主角成长线", "stages": ["起", "承", "转", "合"]},
    {"name": "副线", "description": "感情线", "stages": [...]}
  ],
  "ending": "结局走向",
  "themes": ["成长", "友情", "正义"]
}
```

### 功能 2: 卷纲自动生成
**描述**: 基于总纲 + 目标字数/章节数，LLM 自动拆分为若干卷  
**优先级**: P0  
**触发**: 总纲确定后，用户点击"生成卷纲"  
**逻辑**: 
- 10万字 → 2-3卷
- 30万字 → 5-8卷
- 50万字+ → 8-15卷
- 每卷有独立的小高潮和阶段性目标

### 功能 3: 章纲细化
**描述**: 每卷的章纲包含具体的场景、对话要点、情节转折  
**优先级**: P1  
**触发**: 卷纲确定后，用户点击"细化章纲"  
**数据结构**:
```json
{
  "chapter": 1,
  "title": "家乡变故",
  "scenes": [
    {"location": "张家村", "characters": ["张三", "李四"], "event": "邪派来袭"},
    {"location": "村口", "characters": ["张三", "师父"], "event": "师父出现"}
  ],
  "turning_point": "主角决定离开家乡",
  "emotional_beat": "从平静到震惊到决心"
}
```

### 功能 4: 大纲编辑 UI
**描述**: 前端可视化展示三级大纲，支持编辑  
**优先级**: P0  
**UI**: 树形折叠结构（总纲 → 卷纲 → 章纲）

### 功能 5: 从对话确定总纲
**描述**: 人机交互对话结束后，可将讨论结果直接生成为总纲  
**优先级**: P1  
**流程**: 对话结论 → "生成总纲" → LLM 整理为结构化总纲

## 3. 数据模型变更

### 新增表: outlines

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | 关联小说（唯一） |
| level | VARCHAR(20) | master/volume/chapter |
| volume_number | INTEGER | 卷号（volume/chapter 级别） |
| chapter_number | INTEGER | 章号（chapter 级别） |
| content | JSON | 大纲内容 |
| status | VARCHAR(20) | draft/confirmed |
| updated_at | TIMESTAMPTZ | |

### 修改: volumes 表

volumes.outline 字段保留，但改为引用 outlines 表的 content（或直接从 outlines 读取）。

## 4. API 设计

```
GET    /api/v1/projects/{id}/outlines              — 获取所有大纲（树形）
GET    /api/v1/projects/{id}/outlines/master       — 获取总纲
PUT    /api/v1/projects/{id}/outlines/master       — 编辑总纲
POST   /api/v1/projects/{id}/outlines/generate-volumes  — 基于总纲生成卷纲
POST   /api/v1/projects/{id}/outlines/generate-chapters/{vol_num} — 基于卷纲生成章纲
PUT    /api/v1/projects/{id}/outlines/volume/{num} — 编辑卷纲
PUT    /api/v1/projects/{id}/outlines/chapter/{num} — 编辑章纲
POST   /api/v1/projects/{id}/outlines/from-conversation/{conv_id} — 从对话生成总纲
```

## 5. 验收标准

- [ ] 总纲可手动编辑或从对话生成
- [ ] 基于总纲可自动生成卷纲
- [ ] 基于卷纲可自动生成章纲
- [ ] 前端树形展示三级大纲
- [ ] 每级大纲可独立编辑
- [ ] 生成章节时使用对应章纲作为输入
- [ ] 所有测试通过
