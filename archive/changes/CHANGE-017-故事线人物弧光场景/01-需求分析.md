# 需求分析: 故事线 + 人物弧光 + 场景管理

**变更 ID**: CHANGE-017  
**创建时间**: 2026-05-17  
**优先级**: P1

## 1. 需求背景

当前系统有人物和章节，但缺少它们之间的结构化关联。需要引入：
- **故事线**：贯穿多章的情节线索（主线/副线/暗线）
- **人物弧光**：人物在故事中的成长/转变轨迹
- **场景**：具体的时空场景，关联人物和事件

这三者构成"人物-情节-场景"的三元关系，为后续知识图谱可视化（CHANGE-018）奠定数据基础。

## 2. 核心功能点

### 功能 1: 故事线管理
**描述**: 定义和管理贯穿小说的情节线索  
**数据结构**:
```json
{
  "name": "主线：修仙之路",
  "type": "main|sub|hidden",
  "description": "主角从凡人到仙人的成长历程",
  "key_events": [
    {"chapter": 1, "event": "觉醒灵根", "impact": "开启修仙之路"},
    {"chapter": 5, "event": "拜入宗门", "impact": "获得系统修炼"}
  ],
  "status": "active|resolved|abandoned"
}
```

### 功能 2: 人物弧光
**描述**: 追踪人物在故事中的内在变化  
**数据结构**:
```json
{
  "character_id": 1,
  "arc_type": "growth|fall|flat|transformation",
  "description": "从懦弱少年成长为担当之人",
  "stages": [
    {"chapter_range": [1, 5], "state": "懦弱、依赖他人", "trigger": "家乡被毁"},
    {"chapter_range": [6, 15], "state": "开始独立、学会战斗", "trigger": "师父离去"},
    {"chapter_range": [16, 30], "state": "成熟、有担当", "trigger": "保护同伴"}
  ]
}
```

### 功能 3: 场景管理
**描述**: 定义故事中的重要场景/地点  
**数据结构**:
```json
{
  "name": "青云宗大殿",
  "location": "青云山顶",
  "description": "宗门议事之地，庄严肃穆",
  "appearances": [
    {"chapter": 3, "event": "入门仪式"},
    {"chapter": 20, "event": "宗门大比"}
  ]
}
```

### 功能 4: 三元关联
**描述**: 建立人物↔故事线↔场景的关联关系  
**关联表**:
- storyline_characters: 故事线涉及哪些人物
- storyline_scenes: 故事线发生在哪些场景
- scene_characters: 场景中出现哪些人物

### 功能 5: 前端管理 UI
**描述**: 故事线/弧光/场景的 CRUD 界面  
**位置**: NovelDetail 新增 Tab 或独立页面

## 3. 数据模型

### 新增表

**storylines** (故事线):
| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | |
| name | VARCHAR(200) | 线索名称 |
| type | VARCHAR(20) | main/sub/hidden |
| description | TEXT | 描述 |
| key_events | JSON | 关键事件列表 |
| status | VARCHAR(20) | active/resolved/abandoned |

**character_arcs** (人物弧光):
| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | |
| character_id | FK → characters | |
| arc_type | VARCHAR(30) | growth/fall/flat/transformation |
| description | TEXT | 弧光描述 |
| stages | JSON | 阶段列表 |

**scenes** (场景):
| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | |
| name | VARCHAR(200) | 场景名称 |
| location | VARCHAR(200) | 地理位置 |
| description | TEXT | 描述 |
| appearances | JSON | 出现章节列表 |

**storyline_characters** (故事线-人物关联):
| 字段 | 类型 |
|------|------|
| storyline_id | FK → storylines |
| character_id | FK → characters |
| role_in_line | VARCHAR(50) | 在该线中的角色 |

## 4. API 设计

```
# 故事线
GET/POST   /api/v1/projects/{id}/storylines
GET/PUT/DELETE /api/v1/projects/{id}/storylines/{sl_id}

# 人物弧光
GET/POST   /api/v1/projects/{id}/character-arcs
GET/PUT/DELETE /api/v1/projects/{id}/character-arcs/{arc_id}

# 场景
GET/POST   /api/v1/projects/{id}/scenes
GET/PUT/DELETE /api/v1/projects/{id}/scenes/{scene_id}

# 关联查询
GET /api/v1/projects/{id}/relations  — 获取所有三元关系（供图谱使用）
```
