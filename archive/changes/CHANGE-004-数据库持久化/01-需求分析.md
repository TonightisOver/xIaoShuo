# 需求分析: PostgreSQL 数据库持久化

**变更 ID**: CHANGE-004  
**创建时间**: 2026-05-16  
**优先级**: P0

## 1. 需求背景

xIaoShuo 平台当前使用 JSON 文件存储任务数据，存在并发写入冲突、无事务支持、查询效率低等问题。需要迁移到 PostgreSQL 数据库，支持生产级别的数据持久化。

**目标用户**: 开发团队 / 生产环境  
**使用场景**: 所有任务的 CRUD 操作、并发访问、数据查询

## 2. 核心功能点

### 功能 1: 数据库连接管理
**描述**: 使用 SQLAlchemy 2.0 async 管理数据库连接池  
**优先级**: P0  
**依赖**: 无

**实现要求**:
- 异步连接池（asyncpg 驱动）
- 连接池参数可配置（pool_size, max_overflow）
- pool_pre_ping 保证连接有效性
- 应用启动/关闭时管理连接生命周期

### 功能 2: ORM 模型定义
**描述**: 定义 Task 表的 SQLAlchemy 模型  
**优先级**: P0  
**依赖**: 功能 1

**模型字段**:
- id (主键)、task_id (唯一索引)
- novel_type、title、status
- progress、result、errors (JSONB)
- created_at、updated_at (时间戳)

### 功能 3: 数据库迁移管理
**描述**: 使用 Alembic 管理数据库 schema 版本  
**优先级**: P0  
**依赖**: 功能 2

**实现要求**:
- Alembic 配置和环境文件
- 初始迁移脚本（创建 tasks 表）
- 支持 upgrade/downgrade

### 功能 4: TaskManager 重写
**描述**: 将 TaskManager 从 JSON 文件操作重写为 SQLAlchemy 查询  
**优先级**: P0  
**依赖**: 功能 1, 功能 2

**操作**:
- create_task: INSERT
- get_task: SELECT by task_id
- update_task: UPDATE
- list_tasks: SELECT with filters
- delete_task: DELETE

### 功能 5: 数据迁移脚本
**描述**: 将现有 JSON 文件中的任务数据迁移到 PostgreSQL  
**优先级**: P1  
**依赖**: 功能 2

### 功能 6: 测试数据库配置
**描述**: 配置独立的测试数据库，避免污染开发数据  
**优先级**: P1  
**依赖**: 功能 1

**实现要求**:
- 使用 xiaoshuo_test 数据库
- NullPool 避免事件循环冲突
- 每次测试前清理数据

## 3. 技术可行性分析

### 技术难度
**中等**

- SQLAlchemy 2.0 async 是成熟方案
- asyncpg 是 PostgreSQL 最快的异步驱动
- Alembic 迁移管理是标准实践
- 主要挑战在于 pytest-asyncio 与 asyncpg 的事件循环兼容

### 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 事件循环冲突 | 高 | 测试环境使用 NullPool |
| 连接池泄漏 | 中 | pool_pre_ping + lifespan 管理 |
| 数据迁移丢失 | 中 | 迁移前备份 JSON 文件 |
