## 3. 关键实现细节

### 3.1 NullPool 解决方案

测试中 asyncpg 连接池与 pytest-asyncio 事件循环冲突，表现为 `RuntimeError: Event loop is closed`。解决方案:

```python
engine = create_async_engine(TEST_DATABASE_URL, poolclass=NullPool)
```

NullPool 每次操作创建新连接，不复用连接，彻底避免事件循环生命周期问题。

### 3.2 JSONB 字段处理

progress/result/errors 使用 PostgreSQL JSONB 类型，SQLAlchemy 自动序列化/反序列化 Python dict/list。

### 3.3 时间戳处理

- `created_at`: 创建时设置，不可变
- `updated_at`: 每次更新时自动刷新（server_default + onupdate）
