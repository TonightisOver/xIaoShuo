## 发现的预存在 Bug（非本次变更引入）

**Bug**: `GET /api/v1/projects/{novel_id}/chapters/{chapter_number}` 端点在 FastAPI 响应验证阶段失败。

**原因**: 路由使用 `response_model=ChapterResponse`，而 `ChapterResponse` 要求 `novel_id` 字段，但 `NovelManager.get_chapter()` 返回的字典不包含 `novel_id`。

**错误信息**:
```
fastapi.exceptions.ResponseValidationError: 1 validation error:
  {'type': 'missing', 'loc': ('response', 'novel_id'), 'msg': 'Field required', ...}
```

**影响范围**: 仅影响单章节 GET 端点的 HTTP 响应序列化。服务层逻辑（`get_chapter()` 返回 `volume_number`）完全正确，已通过单元测试和集成测试的服务层调用验证。

**建议修复**: 在 `get_chapter()` 返回字典中加入 `"novel_id": c.novel_id`，或将 `ChapterResponse.novel_id` 改为可选字段。此 Bug 与 CHANGE-029 无关，建议在后续变更中修复。
