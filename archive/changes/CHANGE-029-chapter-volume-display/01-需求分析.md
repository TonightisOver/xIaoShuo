# CHANGE-029 需求分析

## 1. 背景

NovelDetail.vue 的章节 Tab 存在两个独立 bug，均源于前后端数据契约不一致。

---

## 2. 问题 1：章节未按卷分组，出现重复章节号

### 现象
章节 Tab 中多个卷的章节混在一起，出现多个"第1章"、"第2章"，用户无法区分章节归属。

### 根因分析

**数据层（`Chapter` 表）**

`Chapter` 模型（`db_models.py` 第 157-178 行）有两个字段：
- `chapter_number`：全局唯一编号（跨卷连续，如第1卷1-10章，第2卷11-20章）
- `volume_number`：该章节所属卷号（可为 NULL）

但实际生成逻辑存在歧义：
- `generate_volume_background`（`novel_generator.py` 第 656 行）：`chapter_number` 取自 `ch_outline.get("chapter", i+1)`，这个值来自大纲中的 `chapter` 字段，**可能是卷内编号（每卷从1开始）**，也可能是全局编号，取决于 LLM 生成的大纲结构。
- `persist_results`（`novel_generator.py` 第 928 行）：全量生成时章节写入 DB 时**没有设置 `volume_number` 字段**，导致 `volume_number` 为 NULL。

**服务层（`novel_manager.py`）**

`list_chapters`（第 238-248 行）返回的字典**不包含 `volume_number` 字段**：

```python
return [{"id": c.id, "chapter_number": c.chapter_number,
         "title": c.title, "content": c.content,
         "word_count": c.word_count, "status": c.status,
         "updated_at": c.updated_at}
        for c in result.scalars().all()]
```

**前端层（`VolumeList.vue`）**

`volumeChapters` 函数（第 43-46 行）通过 `c.volume_number === vol.volume_number` 过滤章节：

```js
function volumeChapters(vol) {
  const chs = props.chapters.filter(c => c.volume_number === vol.volume_number)
  return chs.length ? chs : null
}
```

由于 API 返回的章节数据中 `volume_number` 字段缺失（`undefined`），`undefined === 1` 永远为 `false`，**所有卷都显示"暂无章节"**。

`NovelDetail.vue` 的 `unassignedChapters` 计算属性（第 311-313 行）：

```js
const unassignedChapters = computed(() =>
  chapters.value.filter(c => !c.volume_number)
)
```

因为 `volume_number` 是 `undefined`（falsy），所有章节都被归入"未分卷章节"，在卷列表下方平铺显示，导致重复章节号问题。

### 问题链路总结

```
list_chapters() 不返回 volume_number
  → 前端 chapters 数组中所有章节的 volume_number 为 undefined
  → VolumeList.volumeChapters() 过滤结果为空 → 每卷显示"暂无章节"
  → unassignedChapters 包含全部章节 → 平铺显示，出现重复章节号
```

## 3. 问题 2：章节编辑页显示"章节不存在"

### 现象
点击章节进入 `/novels/:id/chapters/:num` 后，页面显示"章节不存在"，无法编辑。

### 根因分析

**路由参数**

`ChapterEdit.vue`（第 41 行）：`const chapterNum = route.params.num`

`VolumeList.vue`（第 22 行）链接：`:to="\`/novels/${novelId}/chapters/${ch.chapter_number}\`"`

`NovelDetail.vue`（第 168 行）链接：`:to="\`/novels/${novelId}/chapters/${ch.chapter_number}\`"`

路由参数 `:num` 传入的是 `ch.chapter_number`。

**后端查询**

`get_chapter`（`novel_manager.py` 第 250-264 行）：

```python
select(Chapter).where(
    Chapter.novel_id == novel_id,
    Chapter.chapter_number == chapter_number
)
```

查询条件是 `chapter_number == chapter_number`，即用路由中的数字直接匹配 DB 的 `chapter_number` 字段。

**问题根因**

当 `chapter_number` 是**卷内编号**（每卷从1开始）时：
- 第1卷第1章：`chapter_number = 1`
- 第2卷第1章：`chapter_number = 1`（重复！）

此时 DB 中存在两条 `chapter_number = 1` 的记录，`scalar_one_or_none()` 会抛出 `MultipleResultsFound` 异常，或者查询到错误的章节。

更严重的情况：如果 `generate_volume_background` 写入的 `chapter_number` 是卷内编号（1、2、3...），而 `generate_chapters_background` 写入的是全局编号（11、12、13...），则两套数据的 `chapter_number` 语义不一致，导致部分章节根本查不到。

**"章节不存在"的直接触发路径**

1. 用户点击第2卷的"第1章"，链接为 `/novels/xxx/chapters/1`
2. 后端查 `chapter_number = 1`，找到的是第1卷第1章（或找不到，如果第1卷章节已被覆盖）
3. 如果 `chapter_number` 语义混乱，查询返回 `None` → 404 → 前端显示"章节不存在"

## 4. 验收标准

### 问题 1
- [ ] 章节 Tab 中，每个卷显示为独立分组，卷标题清晰可见
- [ ] 每个卷内只显示属于该卷的章节
- [ ] 章节显示格式为"第N章：标题"，N 为卷内编号（如第1卷第1章显示"第1章"，第2卷第1章也显示"第1章"，但在不同卷分组下）
- [ ] 无卷归属的章节（历史数据）仍在"未分卷章节"区域显示

### 问题 2
- [ ] 点击任意章节均能正确进入编辑页
- [ ] 编辑页能正确加载章节内容
- [ ] 保存、删除、重新生成功能正常工作
- [ ] 同一小说内不同卷的同编号章节（如两个"第1章"）能分别正确访问
