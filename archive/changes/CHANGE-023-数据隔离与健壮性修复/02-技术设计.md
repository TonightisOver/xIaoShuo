# 技术设计: 数据隔离与健壮性修复

**变更 ID**: CHANGE-023  
**阶段**: 02-技术设计

## 1. 数据隔离方案

### 服务层改造

所有子资源的 update/delete 方法增加可选参数 `novel_id`，当提供时在 SQL WHERE 子句追加 `AND novel_id = ?`。

**影响方法**:
- `storyline_service`: update_storyline, delete_storyline
- `storyline_service`: update_character_arc, delete_character_arc
- `storyline_service`: update_scene, delete_scene
- `conversation_service`: 相关查询方法

### 路由层改造

路由处理函数从 URL path 提取 novel_id，传递给 service 方法，确保请求只能操作归属于该小说的资源。

## 2. 错误处理修复

### send_message 顺序修正

**修改前**: 先 insert message → 再查 conversation（失败则 500）  
**修改后**: 先查 conversation 是否存在 → 存在则 insert message

### AI 生成存在性检查

在 AI 生成方法入口处检查 novel 是否存在，不存在则 raise ValueError，由路由层转为 400/404 响应。

## 3. 前端健壮性

StorylineManager.vue 中 AI 生成按钮的异步操作用 try/finally 包裹，确保 loading 状态在异常时也能复位。

## 4. 代码质量

运行 `ruff --fix` 自动修复 import 排序和未使用 import 问题。

## 5. 风险评估

- 风险等级: 低
- 数据隔离为追加 WHERE 条件，不改变现有正常流程
- send_message 顺序调整为纯逻辑重排，无数据结构变更
