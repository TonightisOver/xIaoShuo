## 架构方案

所有隔离逻辑在 **service 层** 强制执行，route 层负责传递 novel_id。不在 route 层做二次校验（避免重复逻辑），service 层是唯一的隔离边界。

## 变更详设

### 1. conversation_service.py

#### get_conversation — 签名变更
```python
# 旧
async def get_conversation(self, conv_id: int) -> dict | None:
    ...where(Conversation.id == conv_id)

# 新
async def get_conversation(self, conv_id: int, novel_id: str) -> dict | None:
    ...where(Conversation.id == conv_id, Conversation.novel_id == novel_id)
```

#### send_message — 传 novel_id
```python
# 旧
async def send_message(self, conv_id: int, content: str) -> dict:
    conv_data = await self.get_conversation(conv_id)

# 新
async def send_message(self, conv_id: int, content: str, novel_id: str) -> dict:
    conv_data = await self.get_conversation(conv_id, novel_id)
    # 后续 get_conversation 调用也传 novel_id
```

#### conclude_conversation — 传 novel_id
```python
# 旧
async def conclude_conversation(self, conv_id: int) -> dict:
    conv_data = await self.get_conversation(conv_id)

# 新
async def conclude_conversation(self, conv_id: int, novel_id: str) -> dict:
    conv_data = await self.get_conversation(conv_id, novel_id)
```

#### confirm_message — 三元组校验
```python
# 旧：只按 msg_id 查
select(MsgModel).where(MsgModel.id == msg_id)

# 新：msg_id + conv_id + conversation.novel_id 三元组
select(MsgModel)
    .join(Conversation, MsgModel.conversation_id == Conversation.id)
    .where(
        MsgModel.id == msg_id,
        MsgModel.conversation_id == conv_id,
        Conversation.novel_id == novel_id,
    )
```

#### generate_outline_from_conv — 内部调用同步
```python
# 内部调用 get_conversation 时传 novel_id（已有 novel_id 参数）
conv_data = await conv_service.get_conversation(conv_id, novel_id)  # 已有 novel_id
```

### 3. storyline_service.py

#### update/delete arc/scene — novel_id 改为必填

```python
# 旧
async def update_character_arc(self, arc_id: int, novel_id: str = None, **kwargs) -> bool:
    query = select(CharacterArc).where(CharacterArc.id == arc_id)
    if novel_id:
        query = query.where(CharacterArc.novel_id == novel_id)

# 新
async def update_character_arc(self, arc_id: int, novel_id: str, **kwargs) -> bool:
    query = select(CharacterArc).where(
        CharacterArc.id == arc_id, CharacterArc.novel_id == novel_id
    )
```

同样处理：`delete_character_arc`、`update_scene`、`delete_scene`、`update_storyline`、`delete_storyline`。

#### add_character_to_storyline — 校验归属

```python
async def add_character_to_storyline(
    self, storyline_id: int, character_id: int,
    novel_id: str, role: str | None = None
) -> None:
    async with get_db_session() as session:
        # 校验故事线属于该小说
        sl = await session.get(Storyline, storyline_id)
        if not sl or sl.novel_id != novel_id:
            raise ValueError("故事线不存在或不属于该小说")
        sc = StorylineCharacter(
            storyline_id=storyline_id,
            character_id=character_id,
            role_in_line=role,
        )
        session.add(sc)
```

#### get_relations — DB 过滤

```python

result = await session.execute(

```

## 关键决策

1. **service 层强制 novel_id，不可选** — 消除"传了就校验，不传就跳过"的歧义，调用方必须明确传递
2. **confirm_message 用 JOIN 而非两次查询** — 原子性校验，避免 TOCTOU 问题
3. **get_relations 改 DB JOIN** — 性能更好，且消除 Python 过滤可能的逻辑错误

## 接口变更影响

| 方法 | 旧签名 | 新签名 | 影响调用方 |
|------|--------|--------|-----------|
| `get_conversation` | `(conv_id)` | `(conv_id, novel_id)` | send_message, conclude_conversation, generate_outline_from_conv, storyline_service.generate_from_conversation |
| `send_message` | `(conv_id, content)` | `(conv_id, content, novel_id)` | conversations route |
| `conclude_conversation` | `(conv_id)` | `(conv_id, novel_id)` | conversations route |
| `update_character_arc` | `(arc_id, novel_id=None, ...)` | `(arc_id, novel_id, ...)` | storylines route |
| `delete_character_arc` | `(arc_id, novel_id=None)` | `(arc_id, novel_id)` | storylines route |
| `update_scene` | `(scene_id, novel_id=None, ...)` | `(scene_id, novel_id, ...)` | storylines route |
| `delete_scene` | `(scene_id, novel_id=None)` | `(scene_id, novel_id)` | storylines route |
| `add_character_to_storyline` | `(sl_id, char_id, role)` | `(sl_id, char_id, novel_id, role)` | storylines route |
