# CHANGE-042 实施方案

## Bug A 修复方案

### 根因确认

`ChapterResponse`（`src/api/models/responses.py` 第 135-145 行）要求 `novel_id: str` 为必填字段：

```python
class ChapterResponse(BaseModel):
    novel_id: str          # 必填，无默认值
    chapter_number: int
    ...
```

但 `NovelManager.get_chapter()`（`src/api/services/novel_manager.py` 第 319-323 行）返回的字典**不包含 `novel_id`**：

```python
return {"id": c.id, "chapter_number": c.chapter_number,
        "volume_number": c.volume_number,
        "title": c.title, "content": c.content,
        "word_count": c.word_count, "status": c.status,
        "updated_at": c.updated_at}
```

FastAPI 用 `ChapterResponse` 序列化时，`novel_id` 缺失，返回 422 Unprocessable Entity。前端 `load()` 函数判断 `!res.ok`，将 `chapter.value` 置为 `null`，触发"章节暂时无法访问"错误页。

章节列表页显示字数正常，是因为 `list_chapters` 不使用 `ChapterResponse` 模型，直接返回原始字典，不经过 Pydantic 验证。

### 修复方案

**方案一（推荐）：在 `get_chapter` 返回值中补充 `novel_id` 字段**

文件：`src/api/services/novel_manager.py`

修改位置：第 319-323 行

```python
# 修改前
return {"id": c.id, "chapter_number": c.chapter_number,
        "volume_number": c.volume_number,
        "title": c.title, "content": c.content,
        "word_count": c.word_count, "status": c.status,
        "updated_at": c.updated_at}

# 修改后
return {"id": c.id, "novel_id": c.novel_id,
        "chapter_number": c.chapter_number,
        "volume_number": c.volume_number,
        "title": c.title, "content": c.content,
        "word_count": c.word_count, "status": c.status,
        "updated_at": c.updated_at}
```

**方案二（备选）：将 `ChapterResponse.novel_id` 改为可选**

文件：`src/api/models/responses.py`

修改位置：第 138 行

```python
# 修改前
novel_id: str

# 修改后
novel_id: str | None = None
```

方案一更正确，因为 `novel_id` 是章节的关键归属字段，应该始终返回，而不是设为可选。

### 附带检查：`update_chapter` 多记录问题

`src/api/services/novel_manager.py` 第 328-334 行，`update_chapter` 使用 `scalar_one_or_none()` 但没有 `.limit(1)`，若同一 `chapter_number` 存在多条记录会抛出 `MultipleResultsFound`。建议同步修复：

```python
# 修改前
result = await session.execute(
    select(Chapter).where(
        Chapter.novel_id == novel_id,
        Chapter.chapter_number == chapter_number
    )
)
ch = result.scalar_one_or_none()

# 修改后
result = await session.execute(
    select(Chapter).where(
        Chapter.novel_id == novel_id,
        Chapter.chapter_number == chapter_number
    ).order_by(Chapter.id.desc()).limit(1)
)
ch = result.scalar_one_or_none()
```

### 涉及文件清单

| 文件 | 行号 | 改动 |
|------|------|------|
| `src/api/services/novel_manager.py` | 319-323 | `get_chapter` 返回值补充 `novel_id` 字段 |
| `src/api/services/novel_manager.py` | 328-334 | `update_chapter` 查询加 `.order_by(Chapter.id.desc()).limit(1)` |

## 压测脚本结构设计

### 目录结构

```
loadtest/
├── locustfile.py          # 主入口，注册所有 TaskSet
├── config.py              # 基础配置（host、novel_id 等）
├── tasks/
│   ├── __init__.py
│   ├── chapter_read.py    # 章节读取场景（高频）
│   ├── chapter_list.py    # 章节列表 + 项目详情场景
│   ├── chapter_generate.py # 章节生成场景（低并发，长耗时）
│   └── ai_rewrite.py      # AI 改写场景
├── data/
│   └── seed.json          # 测试用 novel_id、chapter_number 列表
└── reports/               # 压测报告输出目录（gitignore）
```

### 场景设计

**场景 1：章节读取（高频读）**

文件：`tasks/chapter_read.py`

- 目标接口：`GET /api/v1/projects/{novel_id}/chapters/{chapter_number}`
- 并发用户：50-200 VU
- 持续时间：5 分钟
- 成功标准：P95 响应时间 < 500ms，错误率 < 1%

**场景 2：页面加载（混合读）**

文件：`tasks/chapter_list.py`

模拟用户打开小说详情页，并发调用：
- `GET /api/v1/projects/{novel_id}`
- `GET /api/v1/projects/{novel_id}/chapters`
- `GET /api/v1/projects/{novel_id}/volumes`

- 并发用户：20-100 VU
- 成功标准：P95 < 1s

**场景 3：章节生成（重计算）**

文件：`tasks/chapter_generate.py`

- 目标接口：`POST /api/v1/projects/{novel_id}/generate-chapters`
- 并发用户：3-10 VU（LLM 接口，不宜高并发）
- 持续时间：10 分钟
- 成功标准：任务创建成功率 100%，不触发 409 冲突

**场景 4：AI 改写（LLM 调用）**

文件：`tasks/ai_rewrite.py`

- 目标接口：`POST /api/v1/projects/{novel_id}/chapters/{chapter_number}/rewrite`
- 并发用户：2-5 VU
- 成功标准：P95 < 60s（LLM 响应），504 超时率 < 5%

### `locustfile.py` 结构示例

```python
from locust import HttpUser, between, task
from tasks.chapter_read import ChapterReadTasks
from tasks.chapter_list import ChapterListTasks

class ReadHeavyUser(HttpUser):
    """模拟高频读取用户"""
    wait_time = between(0.5, 2)
    tasks = {ChapterReadTasks: 7, ChapterListTasks: 3}

class GenerateUser(HttpUser):
    """模拟生成操作用户"""
    wait_time = between(30, 120)
    # 低频，避免触发 409
```
