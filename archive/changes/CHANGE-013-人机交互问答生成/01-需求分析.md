# 需求分析: 人机交互问答生成

**变更 ID**: CHANGE-013  
**创建时间**: 2026-05-16  
**优先级**: P1

## 1. 需求背景

当前生成流程是全自动的——用户提交创意后，AI 独立完成所有阶段。用户无法在生成过程中介入、提出修改意见或引导方向。

用户希望在生成前/中可以与 AI 进行对话式讨论：
- 用户提出需求 → AI 提问确认细节 → 用户回答 → AI 再问 → 多轮对话后达成共识
- 用户审核 AI 的方案 → 确认后才开始生成
- 生成过程中可以暂停、修改方向

**目标用户**: 对内容有明确想法但需要 AI 协助细化的创作者  
**使用场景**: 创作前的需求讨论、世界观细化、人物设定讨论、情节走向讨论

## 2. 核心功能点

### 功能 1: 对话式创作会话
**描述**: 用户可以在小说项目中开启一个"创作对话"，与 AI 讨论设定和情节  
**优先级**: P0

**交互流程**:
```
用户: "我想让主角有一个隐藏身份"
AI: "好的，你想让隐藏身份是什么类型？比如：1.前世记忆 2.血脉觉醒 3.组织卧底"
用户: "血脉觉醒，但是要慢慢揭示"
AI: "明白。那揭示的节奏你想怎么安排？建议在第几卷揭示？"
用户: "第二卷末尾有暗示，第三卷正式揭示"
AI: "好的，我整理一下方案：[方案摘要]。确认后我开始生成？"
用户: "确认"
→ 触发生成
```

### 功能 2: 对话历史持久化
**描述**: 对话记录保存在数据库中，可以回顾  
**优先级**: P0

### 功能 3: 对话结论转化为设定
**描述**: 对话中达成的共识可以自动/手动提取为世界观、人物、大纲等设定  
**优先级**: P1

### 功能 4: 前端对话 UI
**描述**: 类似聊天界面，支持多轮对话  
**优先级**: P0

## 3. 数据模型

### 新增表: conversations

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| novel_id | FK → novels | 关联小说 |
| topic | VARCHAR(100) | 讨论主题（世界观/人物/情节/自由） |
| status | VARCHAR(20) | active/concluded |
| created_at | TIMESTAMPTZ | |
| concluded_at | TIMESTAMPTZ | |

### 新增表: messages

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| conversation_id | FK → conversations | |
| role | VARCHAR(20) | user/assistant |
| content | TEXT | 消息内容 |
| created_at | TIMESTAMPTZ | |

## 4. API 设计

```
POST   /api/v1/projects/{novel_id}/conversations          — 创建对话
GET    /api/v1/projects/{novel_id}/conversations          — 列出对话
GET    /api/v1/projects/{novel_id}/conversations/{id}     — 对话详情（含消息）
POST   /api/v1/projects/{novel_id}/conversations/{id}/messages — 发送消息（AI 自动回复）
POST   /api/v1/projects/{novel_id}/conversations/{id}/conclude — 结束对话，提取设定
```

## 5. 非功能需求

- AI 回复使用 DeepSeek API（流式输出可选）
- 对话上下文包含小说已有设定（世界观/人物）
- 单次对话最多 50 轮
- 对话结论可手动编辑后再应用到设定
