# 专家评审报告: 人机交互问答生成

**变更 ID**: CHANGE-013  
**版本**: v1  
**评审时间**: 2026-05-16

## 评审维度

### 1. 架构设计

| 项目 | 评价 | 说明 |
|------|------|------|
| 分层合理性 | 良好 | Route → Service → Model 三层清晰 |
| 职责划分 | 良好 | Service 层封装 LLM 调用和业务逻辑 |
| 扩展性 | 良好 | 后续可升级为流式输出，无需改动模型层 |
| 耦合度 | 良好 | 对话模块独立，仅通过 novel_id 关联 |

### 2. Prompt 工程

| 项目 | 评价 | 说明 |
|------|------|------|
| 上下文注入 | 合理 | 包含世界观/人物/大纲摘要 |
| 历史窗口 | 合理 | 20 轮平衡了上下文质量和 token 消耗 |
| 结论提取 | 合理 | 结构化输出 [{type, content}] 便于后续应用 |
| 角色设定 | 合理 | "资深小说创作顾问"定位清晰 |

### 3. 数据模型

| 项目 | 评价 | 说明 |
|------|------|------|
| 表结构 | 合理 | conversations + messages 两表，关系清晰 |
| 索引 | 合理 | novel_id 和 conversation_id 均有索引 |
| 级联策略 | 合理 | ON DELETE CASCADE 符合业务语义 |

### 4. 前端实现

| 项目 | 评价 | 说明 |
|------|------|------|
| 交互体验 | 良好 | 聊天气泡 UI，自动滚动 |
| 集成方式 | 良好 | NovelDetail Tab 中嵌入，入口自然 |
| 状态管理 | 良好 | 组件内状态，无需全局 store |

## 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| LLM 响应慢导致用户等待 | 低 | 前端有 loading 状态提示 |
| Token 消耗过高 | 低 | 20 轮上下文限制 |
| 结论提取格式不稳定 | 低 | JSON 解析失败有兜底处理 |

## 结论

**通过** — 设计合理，实现完整，风险可控。建议后续迭代考虑流式输出提升体验。
